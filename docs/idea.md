# GraceTree Shorts Studio

말씀기도 쇼츠 자동 제작 시스템 아이디어 문서

## 1. 프로젝트 개요

`GraceTree Shorts Studio`는 매일 제작 중인 말씀기도 유튜브 쇼츠 영상을 자동으로 생성하기 위한 로컬 기반 영상 자동화 프로젝트이다.

현재는 CapCut을 사용해 수동으로 편집하고 있으며, 한 편당 약 30분 정도가 소요된다.  
1차 목표는 YouTube 업로드 자동화가 아니라, CapCut 없이 기존 스타일과 최대한 유사한 `1080x1920` 세로형 mp4 영상을 자동 생성하는 것이다.

이 프로젝트는 처음부터 완전 자동화를 목표로 하지 않는다.  
초기에는 사용자가 매일 필요한 원본 파일과 `script.txt`를 준비하고, 시스템은 해당 파일들을 읽어 정해진 규칙에 따라 최종 영상을 생성한다.

---

## 2. 프로젝트 목표

### 1차 목표

- CapCut 없이 말씀기도 쇼츠 mp4 자동 생성
- 기존 완성본과 최대한 유사한 스타일 구현
- 매일 바뀌는 파일과 공통 리소스를 분리하여 관리
- `script.txt` 기반으로 제목, 말씀, 기도 내용을 읽어 자막 생성
- Whisper를 사용하여 녹음본을 분석하고 자막 싱크 및 구간을 추정
- 최종 결과물로 `final.mp4` 생성

### 2차 목표

1차에서는 구현하지 않고 기록만 남긴다.

- YouTube 예약 업로드 자동화
- YouTube 제목, 설명, 해시태그 자동 적용
- 자막 분할 자동화
- 음량 자동 보정
- 배경음악 볼륨 자동 조정
- 여러 날짜 영상 일괄 생성
- 7일치 또는 30일치 예약 발행
- 썸네일 별도 업로드

## 3. 현재 제작 방식 요약

현재 영상은 다음 요소로 구성된다.

1. 썸네일 이미지 1프레임
2. 제목/말씀 구간용 6초 영상
3. 기도 구간용 18초 영상
4. 스크립트 txt
5. 목소리 녹음본 m4a
6. 배경음악

영상 구조는 다음과 같다.

```text
썸네일 이미지 1프레임
→ 제목/말씀 구간
→ 기도 구간
→ 마지막 여백
→ 배경음악 페이드아웃
```

## 4. 입력 파일 구조

공통 영상 파일은 매일 바뀌지 않으므로 상단 `assets` 폴더에 둔다.  
매일 바뀌는 파일만 날짜별 `input` 폴더에 둔다.

```text
gracetree-shorts-studio/
  assets/
    common/
      intro_6s.mp4
      prayer_loop_18s.mp4

    fonts/
      main.ttf

  input/
    2026-05-25/
      thumbnail.png
      voice.m4a
      bgm.mp3
      script.txt

  output/
    2026-05-25/
      final.mp4
      subtitles.ass
      timing.json
      render_log.txt
```

### 공통 파일

| 파일 | 설명 |
|---|---|
| `intro_6s.mp4` | 제목/말씀 구간에 사용하는 6초 영상 |
| `prayer_loop_18s.mp4` | 기도 구간에 반복 사용하는 18초 영상 |
| `main.ttf` | 자막 렌더링에 사용할 폰트 파일 |

### 날짜별 입력 파일

| 파일 | 설명 |
|---|---|
| `thumbnail.png` | 영상 첫 프레임에 들어갈 썸네일 이미지 |
| `voice.m4a` | 목소리 녹음본 |
| `bgm.mp3` | 배경 찬송 간주음악 |
| `script.txt` | 제목, 말씀, 기도, 자막 내용을 담은 스크립트 |

### 출력 파일

| 파일 | 설명 |
|---|---|
| `final.mp4` | 최종 완성 영상 |
| `subtitles.ass` | 스타일 정보가 포함된 자막 파일 |
| `timing.json` | 구간 및 자막 싱크 정보 |
| `render_log.txt` | 렌더링 로그 |

## 5. script.txt 양식

현재 스크립트는 `[제목]`, `[말씀]`, `[기도]`로 구분되어 있다.  
이 구조는 자동화에 적합하므로 유지한다.

권장 양식은 다음과 같다.

```text
[제목]
타인의 슬픔을
바라볼 때
드리는 기도

[말씀]
롬12:15

즐거워하는 자들과
함께 즐거워하고
우는 자들과 함께 울라

[기도]
하나님,
우리는
타인의 슬픔에

얼마나
무감각할 때가
많은지 모릅니다.

타인의
큰 아픔보다

내 작은 불편을
더 크게 여기며
살아가는
우리의 모습을

주님 아시는 줄
믿습니다.

살아계신
예수님의
이름으로
기도합니다.

아멘.
```

향후 YouTube 업로드 자동화를 고려하면 아래 구간을 추가할 수 있다.

```text
[유튜브]
제목: 타인의 슬픔을 바라볼 때 드리는 기도
설명: 오늘의 말씀기도입니다.
해시태그: #말씀기도 #기도 #성경말씀 #shorts
```

1차에서는 `[유튜브]` 구간을 사용하지 않아도 된다.

## 6. 영상 규격

최종 영상 규격은 다음과 같다.

```text
해상도: 1080x1920
비율: 9:16 세로형
형식: mp4
프레임: 30fps 권장
길이: 보통 1분~1분 30초
```

최종 영상 길이는 기본적으로 녹음본 길이를 기준으로 계산한다.

```text
최종 영상 길이
= 시작 여백 2초
+ 정리된 녹음 길이
+ 종료 여백 3초
```

## 7. 구간 처리 규칙

### 7-1. 시작 구간

- 영상 시작 후 2초 뒤에 목소리가 시작되어야 한다.
- 녹음본 앞부분에 무음이 있으면 자동으로 제거한다.
- 제거 후 앞에 2초 무음을 다시 붙인다.
- 배경음악은 영상 시작부터 재생된다.
- 배경음악은 시작 시 2초 페이드인 처리한다.

### 7-2. 썸네일 이미지

- `thumbnail.png`는 영상 맨 앞에 1프레임만 삽입한다.
- 재생 중에는 거의 보이지 않지만 YouTube 썸네일 후보로 잡히게 하는 목적이다.
- 이미지는 1080x1920 세로형을 권장한다.

### 7-3. 제목/말씀 구간

- `intro_6s.mp4` 하나가 제목과 말씀 전체 구간에 사용된다.
- 제목/말씀 구간 길이는 Whisper 분석 결과를 기반으로 추정한다.
- 제목/말씀 구간이 6초보다 길면 `intro_6s.mp4`를 느리게 늘린다.
- 제목/말씀 구간이 6초보다 짧으면 `intro_6s.mp4`를 빠르게 줄인다.
- 기존 작업 기준으로 속도 조절 시 어색함은 크지 않았다.

### 7-4. 기도 구간

- `prayer_loop_18s.mp4`를 기도 길이에 맞춰 반복한다.
- 18초 영상을 처음부터 끝까지 반복한다.
- 반복 사이에는 0.5초 크로스페이드 효과를 적용한다.
- CapCut의 “혼합” 효과와 유사하게 두 영상이 겹쳐 부드럽게 넘어가면 된다.
- 일반 명칭은 `Crossfade` 또는 `Cross Dissolve`로 본다.

### 7-5. 마지막 구간

- 마지막 “아멘” 음성이 끝난 뒤 약 3초 후 영상이 종료된다.
- “아멘” 자막은 약 2초 정도 유지한다.
- 마지막 자막에는 0.5초 페이드아웃을 적용한다.
- 배경 영상은 음성 종료 후 약 3초 더 유지한다.
- 배경음악은 마지막 2초 동안 페이드아웃한다.

## 8. 자막 처리 규칙

자막은 단순 텍스트가 아니라 위치, 크기, 테두리, 그림자, 페이드 효과가 필요하다.  
따라서 SRT보다 ASS 자막 형식을 사용하는 것이 적합하다.

### 용어 설명

```text
SRT
= 자막 시간과 텍스트만 담는 단순 자막 파일

ASS
= 자막 시간, 폰트, 크기, 위치, 테두리, 그림자, 효과까지 담을 수 있는 자막 파일
```

### 자막 스타일

| 구간 | 위치 | 크기 | 색상 | 효과 |
|---|---|---:|---|---|
| 제목 | 상단 | 약 22 | 흰색 | 페이드아웃 |
| 말씀 | 상단 | 약 16 | 흰색 | 페이드인, 페이드아웃 |
| 기도 | 중앙 | 약 24 | 흰색 | 검은색 테두리, 그림자, 일부 페이드 |

### 기도 자막 규칙

- 기도 자막은 중앙에 표시한다.
- 최소 2줄, 최대 4줄 정도로 표시된다.
- 자막 블록 분할은 1차에서는 사용자가 `script.txt`에서 직접 줄바꿈으로 지정한다.
- 데이터가 쌓이면 추후 자동 분할 가능성을 검토한다.
- 문장 사이 전환은 별도 효과 없이 바로 바뀐다.
- 기도 시작 자막에는 페이드인을 적용한다.
- 기도 마지막 자막에는 페이드아웃을 적용한다.

## 9. Whisper 사용 방식

Whisper는 녹음본을 분석하여 텍스트와 시간 정보를 추출하는 음성인식 도구이다.

1차에서는 Whisper를 다음 목적으로 사용한다.

```text
voice.m4a 분석
→ 음성 텍스트 추출
→ 시간 정보 추출
→ script.txt와 비교
→ 제목/말씀/기도 구간 추정
→ 자막 싱크 초안 생성
```

### 주의 사항

녹음본 안에서 제목, 말씀, 기도 구간이 항상 명확한 고정 문장으로 구분되어 있지는 않다.

예를 들어 다음과 같은 고정 기준이 없다.

- “기도 시작” 같은 문장 없음
- “기도하겠습니다” 같은 고정 멘트 없음
- 제목, 말씀, 기도가 항상 명확한 문장 패턴으로 나뉘지 않음

따라서 Whisper 결과는 100% 확정값으로 사용하지 않고, 1차에서는 추정값으로 사용한다.

시스템은 다음처럼 판단할 수 있어야 한다.

```json
{
  "intro_start": 2.0,
  "intro_end": 17.8,
  "prayer_start": 18.0,
  "prayer_end": 70.5,
  "confidence": "high"
}
```

확신도가 낮으면 자동 렌더링을 중단하고 로그에 확인이 필요하다고 남긴다.

```text
기도 구간 시작점을 정확히 찾지 못했습니다.
script.txt의 기도 첫 문장과 녹음본 내용이 다를 수 있습니다.
확인이 필요합니다.
```

## 10. 음량 처리

1차에서는 음량 자동 보정 기능을 제외한다.

현재 작업 방식도 별도 음량 보정 없이 받은 파일 그대로 사용하고 있다.  
따라서 1차에서는 다음만 처리한다.

- 배경음악 시작 2초 페이드인
- 배경음악 마지막 2초 페이드아웃
- 목소리와 배경음악을 합성

다음 기능은 추후 검토한다.

- 목소리 음량 자동 보정
- 배경음악 볼륨 자동 조정
- 목소리가 나올 때 배경음악을 자동으로 낮추는 기능

## 11. 폰트 처리

현재 CapCut에서는 `카페24단정해` 폰트를 사용하고 있다.  
자동 렌더링에서 동일한 폰트를 사용하려면 폰트 파일이 필요하다.

1차 기준은 다음과 같다.

```text
1순위: 카페24단정해 폰트 파일 확보 후 사용
2순위: 비슷한 무료 폰트로 대체
3순위: 임시로 Pretendard, SUITE 등 안정적인 한글 폰트 사용
```

기존 CapCut 영상과 최대한 동일하게 보이려면 폰트 확보가 중요하다.

## 12. 구현 도구 후보

### Python

전체 자동화 흐름을 제어하는 언어로 사용한다.

역할:

- 입력 폴더 확인
- `script.txt` 파싱
- Whisper 실행
- 구간 시간 계산
- 자막 파일 생성
- FFmpeg 명령 실행
- 결과물 저장
- 로그 생성

### FFmpeg

영상과 오디오를 실제로 처리하는 도구이다.

역할:

- 영상 속도 조절
- 영상 반복
- 크로스페이드 적용
- 썸네일 1프레임 삽입
- 자막 합성
- 배경음악 합성
- 최종 mp4 출력

### Whisper

녹음본을 분석하는 도구이다.

역할:

- m4a 음성 분석
- 말한 내용 추출
- 시간 정보 추출
- 자막 싱크 초안 생성
- 제목/말씀/기도 구간 추정

## 14. 실행 흐름

예상 실행 흐름은 다음과 같다.

```text
1. 사용자가 input/YYYY-MM-DD/ 폴더에 파일을 넣는다.
2. Python 스크립트를 실행한다.
3. script.txt를 읽어 제목, 말씀, 기도를 분리한다.
4. voice.m4a의 앞 무음을 제거한다.
5. 앞에 2초 무음을 붙인다.
6. Whisper로 voice.m4a를 분석한다.
7. script.txt와 Whisper 결과를 비교해 제목/말씀/기도 구간을 추정한다.
8. intro_6s.mp4를 제목/말씀 구간 길이에 맞춘다.
9. prayer_loop_18s.mp4를 기도 구간 길이에 맞춰 반복한다.
10. 반복 구간 사이에 0.5초 크로스페이드를 적용한다.
11. subtitles.ass를 생성한다.
12. 배경음악을 합성하고 2초 페이드인/페이드아웃을 적용한다.
13. 썸네일 이미지를 첫 1프레임에 넣는다.
14. final.mp4를 output/YYYY-MM-DD/에 저장한다.
15. 처리 결과를 render_log.txt에 저장한다.
```

## 16. 개발 시 중요한 기준

이 프로젝트는 화려한 관리자 페이지보다 안정적인 자동 생성이 중요하다.

우선순위는 다음과 같다.

```text
1. 기존 영상과 비슷한 결과물
2. 매일 반복 사용 가능한 단순한 폴더 구조
3. 실패 시 원인을 확인할 수 있는 로그
4. 사람이 수정 가능한 중간 파일 생성
5. 이후 YouTube 자동 업로드로 확장 가능한 구조
```

## 17. 코덱스/클로드 작업 지시 방향

이 프로젝트를 AI 개발 도구에 맡길 때는 한 번에 전체 구현을 요청하지 않는다.  
다음 순서로 나누어 진행한다.

### Step 1. 프로젝트 스캐폴딩

- Python 프로젝트 구조 생성
- `assets`, `input`, `output` 폴더 구조 생성
- 설정 파일 생성
- 샘플 `script.txt` 파서 작성

### Step 2. script.txt 파서

- `[제목]`, `[말씀]`, `[기도]` 구간 파싱
- 성경 본문 표기 처리
- 빈 줄 기준 자막 블록 분리
- 파싱 결과 JSON 출력

### Step 3. Whisper 연동

- `voice.m4a` 분석
- 텍스트 및 시간 정보 추출
- 분석 결과 저장
- `timing.json` 초안 생성

### Step 4. 자막 생성

- `subtitles.ass` 생성
- 제목, 말씀, 기도별 스타일 분리
- 페이드인/페이드아웃 적용
- 기도 자막 중앙 배치

### Step 5. FFmpeg 렌더링

- intro 영상 속도 조절
- prayer 영상 반복
- 0.5초 크로스페이드
- 배경음악 합성
- 썸네일 1프레임 삽입
- 최종 mp4 생성

### Step 6. 보정

- 기존 CapCut 완성본과 비교
- 자막 위치, 크기, 그림자, 테두리 보정
- 영상 전환 효과 보정
- 결과물 품질 개선
