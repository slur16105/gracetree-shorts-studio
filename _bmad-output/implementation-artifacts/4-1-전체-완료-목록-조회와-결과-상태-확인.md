# Story 4.1: 전체 완료 목록 조회와 결과 상태 확인

Status: done

## Story

As a 개인 제작자,
I want 완료된 게시 날짜 전체를 한눈에 확인하고 싶다,
so that 어디까지 제작했는지 빠르게 파악할 수 있다.

## Acceptance Criteria

1. 앱 시작 시 SQLite 완료 목록을 게시 날짜 내림차순으로 불러오고 제목·게시 날짜·실제 생성일·결과 폴더 상태를 표시한다.
2. 이력이 없으면 새 작업 등록과 첫 작업 안내 빈 상태를 표시한다.
3. 긴 제목은 한 줄 말줄임표와 전체 제목 tooltip/accessibility name을 제공한다.
4. 완료 행 선택은 최근 작업 정보를 갱신하며 선택만으로 폴더를 열지 않는다.
5. 시작·새로고침 시 실제 결과 경로를 대조하고 없으면 `결과 폴더를 찾을 수 없습니다`로 표시하며 정상 open 상태로 두지 않는다.
6. Tab과 Enter/Space로 행 선택과 폴더 열기를 별도 조작하고 포커스와 선택을 색상 외 방식으로 구분한다.

## Dev Notes

- SQLite 완료 기록이 목록의 기준이고 파일 존재 대조가 열기 가능 상태를 결정한다. 파일 누락 시 기록을 자동 삭제하지 않는다.
- 행 자체는 선택 동작, 내부 버튼은 폴더 열기 동작이다. event propagation으로 선택과 열기가 동시에 실행되지 않게 한다.
- 기본 정렬은 게시 날짜 내림차순이다. 실제 생성일 정렬로 바꾸지 않는다.
- 별도 완료 목록 전용 화면을 만들지 않고 홈 오른쪽 영역을 확장한다.
