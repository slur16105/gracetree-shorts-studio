# Story 2.17: 외부 배포 서명 게이트

Status: review

## Story

As a 배포 담당자,
I want 외부 배포 전에 플랫폼 신뢰 요건을 확인하고 싶다,
so that 사용자가 검증 가능한 설치본을 받을 수 있다.

## Acceptance Criteria

1. 외부 배포 후보에서 Windows code signing과 macOS signing·notarization 상태를 확인하고 개인 개발 빌드와 외부 배포 빌드를 구분하며 자동 업데이트는 제외한다.

## Tasks / Subtasks

- [x] release channel을 `development`와 `external`로 명시하고 artifact metadata에 기록한다. (AC: 1)
- [x] Windows installer·실행 파일의 서명 체인과 timestamp를 검증하는 gate를 추가한다. (AC: 1)
- [x] macOS app/installer의 code signature, hardened runtime, notarization, stapling을 검증한다. (AC: 1)
- [x] secret은 CI secret store에서만 사용하고 로그·artifact에 노출하지 않는다. (AC: 1)
- [x] unsigned/unnotarized external build를 publish하지 못하게 하고 개발 build는 명확히 구분한다. (AC: 1)
- [x] 자동 업데이트 기능·feed·권한을 추가하지 않는다. (AC: 1)

## Dev Notes

- 인증서·Apple/Windows 계정 준비는 외부 권한이 필요한 운영 prerequisite다. 없으면 gate 구현은 가능하지만 실제 external pass는 blocked로 보고한다.
- 서명 실패를 무시하거나 “개발용이므로 성공”으로 바꾸지 않는다.
- notarization은 macOS 외부 배포 산출물에 적용하고 결과 ticket/staple 증거를 보존한다.
