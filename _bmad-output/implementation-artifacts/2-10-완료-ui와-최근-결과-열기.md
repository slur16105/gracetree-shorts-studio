# Story 2.10: 완료 UI와 최근 결과 열기

Status: done

## Story

As a 개인 제작자,
I want 생성 직후 완료 결과를 확인하고 폴더를 열고 싶다,
so that 최종 영상을 바로 사용할 수 있다.

## Acceptance Criteria

1. 결과 커밋 성공 시 성공 모달과 최근 작업에 제목·게시 날짜·실제 생성일·결과 위치를 표시하고 현재 세션 최소 완료 행을 즉시 갱신한다.
2. 결과 폴더 열기는 저장된 작업 ID와 경로를 재검증한 뒤 Explorer/Finder로 열며 renderer 임의 경로와 앱 내부 재생·삭제를 허용하지 않는다.
3. 경로가 없거나 관리 루트 밖이면 OS 호출을 막고 행동 가능한 오류를 표시한다.

## Tasks / Subtasks

- [x] completed event를 완료 DTO와 recent job store projection으로 변환한다. (AC: 1)
- [x] Result Dialog와 최소 Completion Row를 구현하고 포커스 진입·복원을 지원한다. (AC: 1)
- [x] `openResultFolder(jobId)`만 preload에 노출하고 main이 DB 경로·존재·관리 루트를 재검증한다. (AC: 2, 3)
- [x] 검증 후 Electron `shell.openPath`를 호출하고 반환 오류를 AppError로 매핑한다. (AC: 2, 3)
- [x] 성공 UI, 경로 누락·이탈·symlink, 임의 renderer 경로 거부 테스트를 추가한다. (AC: 1~3)

## Dev Notes

- renderer는 경로를 argument로 전달하지 않는다. jobId만 전달하고 main/Python 저장소가 authoritative path를 조회한다.
- 행 선택과 폴더 열기 버튼은 별도 동작이다. 앱 내부 영상 플레이어·삭제는 MVP 비목표다.
- 완료 UI는 Story 4.1 전체 이력의 최소 기반만 제공한다.
