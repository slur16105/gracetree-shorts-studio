# Story 2.11: 게시 가능 품질 게이트

Status: ready-for-dev

## Story

As a 제품 책임자,
I want 기준 영상과 생성 결과의 품질을 검증하고 싶다,
so that 자동화 결과가 실제 게시에 적합한지 판단할 수 있다.

## Acceptance Criteria

1. 최소 10개 기준 작업에서 자막 위치·한글·타이밍·전환·검은 프레임·음성·BGM을 자동·수동 평가하고 8개 이상이 추가 편집 없이 게시 가능해야 통과한다.
2. 실패 작업의 원인 범주와 보정 항목을 릴리스 품질 보고서에 남기고 결정적 자동 테스트와 장기 품질 게이트를 구분한다.

## Tasks / Subtasks

- [ ] 개인정보·라이선스가 정리된 최소 10개 대표 작업과 평가 rubric을 고정한다. (AC: 1)
- [ ] 자동 probe·frame·subtitle·audio 검사 결과를 작업별로 수집한다. (AC: 1)
- [ ] 수동 시청·청취 평가 양식과 pass/fail 근거를 작성한다. (AC: 1, 2)
- [ ] 게시 가능 8/10 gate와 원인 taxonomy·보정 backlog를 릴리스 보고서에 기록한다. (AC: 1, 2)
- [ ] 자동 regression 실패와 품질 표본 실패가 각각 어떤 release gate를 막는지 명시한다. (AC: 2)

## Dev Notes

- 이 Story는 기능 코드를 임의 조정하는 작업이 아니라 검증과 결과 기록이다. 보정이 필요하면 별도 승인 Story/변경 제안으로 넘긴다.
- 평가자, 날짜, 빌드 ID, 플랫폼, fixture checksum을 기록한다.
- “게시 가능” 판단과 자동 테스트 통과를 동일시하지 않는다.
