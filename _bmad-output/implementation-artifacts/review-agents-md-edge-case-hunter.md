# Edge Case Hunter Review Prompt

저장소 루트 `AGENTS.md`와 관련 프로젝트 문서를 읽고 모든 분기·경계 조건을 검토하라.

중점:

- 지침 우선순위가 충돌하거나 순환하는 경우
- 로컬 문서가 없거나 오래된 경우
- 기존 dirty worktree, 생성 파일, 마이그레이션, 파괴적 작업의 경계
- “최소 변경”, “위험도에 맞게”, “필요성이 입증될 때”처럼 서로 다르게 해석될 표현
- 프로젝트 Architecture와 실제로 충돌하는 규칙
- 지침 때문에 정상 작업이 불필요하게 중단되거나 위험 작업이 승인 없이 진행되는 경로

보고 형식:

- 심각도
- 정확한 파일·문장
- 재현 가능한 실패 시나리오
- 최소 수정안

처리되지 않은 실질적 edge case만 보고하라. 스타일 취향은 제외한다.
