# Story 1.2: 안전하고 접근 가능한 데스크톱 앱 셸

Status: ready-for-dev

## Story

As a 개인 제작자,
I want Windows와 macOS에서 실행되는 안전한 데스크톱 앱 셸을 사용하고 싶다,
so that 영상 제작 기능을 일관된 로컬 환경에서 사용할 수 있다.

## Acceptance Criteria

1. **Given** 사용자가 앱을 실행했을 때
   **When** 메인 창이 표시되면
   **Then** 홈, 사용 가이드, 설정 진입점이 있는 기본 앱 셸이 표시된다.
   **And** 1180×720의 권장 레이아웃과 다크 디자인 토큰이 적용된다.

2. **Given** renderer가 운영체제 기능을 사용해야 할 때
   **When** 파일이나 프로세스 기능에 접근하려 하면
   **Then** renderer는 Node.js API에 직접 접근할 수 없다.
   **And** `contextIsolation`, sandbox 및 로컬 리소스 전용 CSP가 활성화된다.

3. **Given** renderer와 main 프로세스가 통신할 때
   **When** preload API를 사용하면
   **Then** 타입이 지정된 허용 API만 노출된다.
   **And** 임의 파일 시스템 및 프로세스 실행 기능은 노출되지 않는다.

4. **Given** 사용자가 키보드만 사용할 때
   **When** `Tab`, `Enter`, `Space`로 기본 앱 셸을 탐색하면
   **Then** 모든 전역 진입점에 접근할 수 있다.
   **And** 현재 포커스와 활성 화면이 색상 외의 방식으로도 구분된다.

5. **Given** 앱 셸의 상태와 선택 지점을 표시할 때
   **When** 디자인 토큰을 적용하면
   **Then** 브랜드 분홍색은 선택, 녹색은 준비·진행·성공, 경고와 위험 색상은 확인 필요·실패 의미로 사용한다.
   **And** 일반 텍스트 4.5:1, 큰 텍스트와 비텍스트 UI 3:1 이상의 목표 대비를 유지한다.

6. **Given** 운영체제에서 모션 감소가 활성화되어 있을 때
   **When** 앱 셸과 상태 피드백을 표시하면
   **Then** 반복 펄스와 불필요한 이동 애니메이션을 줄인다.
   **And** 상태 의미를 애니메이션에만 의존하지 않는다.

7. **Given** 창이 1180×720의 권장 크기보다 작아질 때
   **When** 핵심 레이아웃을 표시하면
   **Then** 모바일 스택으로 재배치하지 않는다.
   **And** 주요 버튼과 진행률이 잘리지 않도록 최소 창 크기를 제한하거나 안내한다.

8. **Given** 네트워크 연결이 없을 때
   **When** 앱을 실행하고 기본 화면을 탐색하면
   **Then** 앱 셸과 로컬 기능은 정상 동작한다.
   **And** 외부 연결을 요구하는 오류나 배너를 표시하지 않는다.

9. **Given** Windows 또는 macOS의 디스플레이 배율이 200%일 때
   **When** 홈, 가이드 및 설정 화면을 탐색하면
   **Then** 핵심 정보와 주요 동작을 확대·스크롤을 통해 사용할 수 있다.
   **And** 버튼, 포커스 표시 및 상태 문구가 겹치거나 잘리지 않는다.

## Dev Notes

### Developer Context

- 이 Story는 “페이지별 전체 UI”를 완성하는 작업이 아니라 모든 후속 화면이 공유할 데스크톱 shell, 보안 경계, 디자인 토큰과 접근성 기반을 만든다.
- Story 1.1은 현재 `ready-for-dev`이며 아직 코드가 구현되지 않았다. 따라서 이 Story의 UPDATE 대상 파일은 현시점에 존재하지 않는다. 개발 에이전트는 Story 1.1 완료 후 실제 생성된 파일을 전부 읽고 아래 예상 경로를 실제 구조에 맞춰 확장해야 한다.
- Story 1.7이 공통 리소스 설정 내용과 교체 기능을, Story 1.8이 사용 가이드의 실제 콘텐츠를 소유한다. 이 Story에서는 진입점과 shell/empty state만 만든다.
- Story 1.3 이후의 날짜 선택, 파일 등록, 완료 목록, 작업 상태를 가짜 데이터로 채우지 않는다. 후속 Story가 안정적으로 삽입될 수 있는 semantic region과 레이아웃 경계만 제공한다.
- UX 참고 시안의 Direction A “Studio Black”을 사용하되, 시안의 샘플 작업·완료 목록·달력·진행률은 이 Story의 구현 데이터가 아니다.

### Previous Story Intelligence

- Story 1.1이 main/preload/renderer 구조, pnpm workspace, contracts 패키지, 테스트와 CI를 소유한다.
- Story 1.1에서 금지한 renderer의 Node/파일/프로세스 직접 접근을 그대로 유지한다.
- 공유 공개 타입은 `packages/contracts`에 두고 앱 내부에서 별도 계약을 재정의하지 않는다.
- Story 1.1의 JSON Lines health contract와 Python engine은 이 Story에서 UI 기능으로 노출하거나 확장하지 않는다.
- Story 1.1 테스트와 frozen lockfile CI는 회귀 검증 대상이다. UI 테스트 추가를 이유로 기존 계약 테스트를 비활성화하지 않는다.

### Technical Requirements

- BrowserWindow security:
  - `nodeIntegration: false`
  - `contextIsolation: true`
  - `sandbox: true`
  - `webSecurity`를 비활성화하지 않음
  - remote navigation/new window를 기본 거부
  - `<webview>`를 사용하지 않음
- renderer는 로컬 번들 자원만 사용한다. 외부 CDN, 원격 폰트, analytics, telemetry, network status banner를 추가하지 않는다.
- preload는 메서드 단위의 좁은 bridge만 허용한다. 원시 IPC 송신기, channel 문자열 입력, Node/Electron 객체를 노출하지 않는다.
- 최소 창 크기는 1180×720을 보존한다. 더 작은 viewport에서 모바일 세로 stack으로 바꾸지 않는다.
- 200% 확대에서는 콘텐츠 스크롤을 허용하되 전역 진입점, 상태 영역, dialog의 핵심 동작을 접근할 수 있어야 한다.
- `Tab`은 논리적 DOM 순서를 따르고 `Enter`/`Space`는 native button 동작을 사용한다. 클릭 가능한 `div`에 키보드 동작을 수동 복제하지 않는다.
- 상태 의미는 색상이나 모션 하나에만 의존하지 않는다.

### Architecture Compliance

- renderer view 전환은 React Router가 아니라 작은 앱 수준 view state로 구현한다.
- UI 상태 라이브러리 Zustand 5.0.14 계열은 Architecture에 승인되어 있지만, 이 Story의 shell 상태가 React local state로 충분하면 store를 선행 도입하지 않는다. 실제 공유 상태가 필요한 Story에서 승인 버전을 사용한다.
- 전역 토큰은 CSS custom properties, 기능/컴포넌트 스타일은 CSS Modules를 사용한다.
- renderer는 main 또는 Python 모듈을 import하지 않는다.
- `desktopApi` 공개 계약은 `packages/contracts/src/desktop-api.ts`가 소유한다.
- 홈, 가이드, resources는 향후 `renderer/src/features/` 경계로 확장하되 shell 전용 primitive만 `shared/components`에 둔다.

### UX / Visual Requirements

- 기준 디자인: GraceTree Studio Black.
- 전역 색상:
  - base `#080909`, nav `#090A0A`, panel `#0D0F0F`, raised `#171919`
  - primary text `#F4F5F5`, secondary text `#9FA4A4`
  - selection/brand `#FF2F62`, success `#36D399`, warning `#F2B84B`, danger `#F05D5E`, focus `#7DD3FC`
- 레이아웃:
  - sidebar 78px
  - bottom status 70px
  - desktop horizontal composition
  - 권장 최소 콘텐츠 1180×720
- sidebar button:
  - 44×44px
  - 홈, 사용 가이드, 설정 순서
  - 설정은 sidebar 하단
  - 활성 상태와 keyboard focus를 별도 표현
- 브랜드 분홍색은 선택 지점, 녹색은 준비·진행·성공에만 사용한다. 현재 shell의 빈 상태를 성공색으로 장식하지 않는다.
- icon glyph 문자만으로 의미를 전달하지 말고 accessible name과 상태를 제공한다.
