# Story 1.8: 작업 흐름을 설명하는 사용 가이드

Status: ready-for-dev

## Story

As a 개인 제작자,
I want 앱 안에서 파일 준비와 오류 해결 방법을 확인하고 싶다,
so that 별도 영상 편집 지식이나 외부 문서 없이 첫 작업을 완료할 수 있다.

## Acceptance Criteria

1. 가이드에 첫 영상 만들기, 파일명 규칙, 스크립트 작성법, 오류 해결, 저장 위치, 앱 정보 섹션과 현재 탐색 상태를 제공한다.
2. 첫 영상 만들기는 게시 날짜, 파일 등록·검증, 생성·결과 확인의 3단계를 짧은 Guide Card로 설명한다.
3. `voice.*`, `bgm.*` 규칙과 `[제목]`, `[말씀]`, `[기도]` 필수 구역, 수정 예시를 제공한다.
4. 주요 오류별 조치와 날짜별 입력·결과·로그 위치를 설명하되 플랫폼 경로를 하드코딩하지 않는다.
5. 홈 복귀 시 선택 날짜, 슬롯, 준비 상태를 보존하고 가이드 이동이 저장 데이터나 실행 스냅샷을 바꾸지 않는다.
6. 논리적 제목·읽기 순서와 활성 섹션·홈 복귀의 접근성 이름을 제공한다.

## Tasks / Subtasks

- [ ] Story 1.2의 guide empty state를 실제 guide feature로 교체한다. (AC: 1~6)
- [ ] 섹션 navigation과 Guide Card를 semantic heading/list/article 구조로 구현한다. (AC: 1, 2, 6)
- [ ] 현재 구현 계약과 일치하는 파일명·스크립트·오류·저장 위치 콘텐츠를 작성한다. (AC: 2~4)
- [ ] 저장 위치는 `desktopApi`가 제공하는 표시용 루트 이름 또는 플랫폼 중립 문구만 사용한다. (AC: 4)
- [ ] view 전환 시 job-editor Zustand 상태와 실행 스냅샷이 보존되는지 확인한다. (AC: 5)
- [ ] 키보드·스크린리더·콘텐츠 계약 테스트를 추가한다. (AC: 1~6)

## Dev Notes

- 가이드는 현재 구현된 MVP 동작만 설명한다. 타임라인 편집, 앱 내부 재생·삭제, 자동 재시도, YouTube 업로드를 암시하지 않는다.
- 오류 코드는 보조 정보이며 사용자가 취할 행동을 우선한다.
- Windows drive letter나 macOS 절대 경로를 문서에 넣지 않는다. `게시 날짜 작업 폴더의 input/output/logs`처럼 구조를 설명한다.
- 정적 콘텐츠를 위한 새 router, CMS, markdown runtime 의존성을 추가하지 않는다.
