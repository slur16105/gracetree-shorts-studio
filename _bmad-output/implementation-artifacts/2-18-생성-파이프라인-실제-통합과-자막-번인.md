# Story 2.18: 생성 파이프라인 실제 통합과 자막 번인

Status: ready-for-dev

## Story

As a 개인 제작자,
I want 생성을 시작하면 음성 정렬부터 자막 번인·배경 합성·최종 영상까지 실제로 실행되기를 원한다,
so that 64×64 자리표시자가 아니라 게시 가능한 1080×1920 자막 쇼츠가 결과로 나온다.

## 배경 (왜 이 스토리가 필요한가)

Epic-2의 합성 스토리(2.5 자막, 2.6 배경, 2.7 합성, 2.8 통합검증, 2.9 최종검증)는 각 모듈이
구현·단위테스트까지 완료됐으나, **orchestrator의 실제 생성 흐름에 연결되지 않았다.** 현재
`_run_start_job_stages`는 `speech_alignment` 뒤에 `vertical_slice` 단계로
`color=black:size=64x64:duration=2` 자리표시자 MP4만 생성한다(orchestrator.py:430~499).
그 결과 어떤 입력을 넣어도 2초짜리 검은 64×64 영상이 "completed"로 기록된다.

추가로, **자막 번인(burn-in)이 코드 어디에도 없다**: `generate_subtitles()`가 `subtitles.ass`를
쓰지만 `compose_background`/`compose_video_audio` 어느 ffmpeg 명령도 `subtitles=`/`ass=` 필터로
영상에 굽지 않는다. `validate_final_artifacts`는 `.ass` 파일 존재만 검사한다. 즉 통합만 해도
자막이 안 보이는 영상이 나온다.

## Acceptance Criteria

1. 생성 시작 시 orchestrator가 `speech_alignment → subtitle_generation → background_composition → final_composition → artifact_validation → commit` 순서로 실제 합성 모듈을 호출하고, 각 단계마다 `stage_started`/`progress` 이벤트와 취소 체크포인트를 둔다. `vertical_slice` 자리표시자 단계는 제거한다.
2. 자막이 최종 영상에 실제로 번인된다. `compose_video_audio`(및 `build_compose_cmd`)가 `subtitles.ass`를 입력받아 libass(`subtitles=`/`ass=` 필터)로 배경 영상 위에 굽고, 한글 글리프가 정상 렌더링되도록 폰트를 지정한다.
3. 한글 폰트가 보장된다. 앱에 OFL 등 재배포 가능한 한글 폰트를 번들하고 libass `fontsdir`/`fontfile`로 지정한다. 공통 리소스 `subtitle_font`는 **선택적 덮어쓰기**로 바꾼다(미설정 시 번들 기본 폰트 사용). UI readiness의 필수 목록에서 `subtitle_font`를 제거한다.
4. 최종 산출물이 `1080×1920`, 30fps, 영상+오디오 스트림을 갖고 `validate_final_artifacts`를 통과한다. 검증 성공 후에만 `final.mp4`·`subtitles.ass`·`timing.json`·`render_log.txt`를 `output/`에 원자적으로 저장하고, temp를 정리한 뒤 단 하나의 `completed` 이벤트를 보낸다(Story 2.9 동작).
5. 단계별 실패는 해당 stageId와 redact된 진단으로 `job_failed`를 보내고, 원본 입력·기존 결과·DB 무결성을 보존한다(부분 산출물이 최종 경로에 남지 않음).
6. 실제 라이선스 안전 샘플(또는 사용자 제공 샘플)로 end-to-end 실행 시 자막이 보이는 1080×1920 MP4가 생성됨을 통합 테스트로 검증한다.

## Dev Notes

### 합성 모듈 인터페이스 (audit 결과, 2026-06-27)

데이터 흐름은 일관됨 — 모두 `attempt_dir`에 출력, `timing.json`을 공유:

- `subtitles.generator.generate_subtitles(script_ast, timing, attempt_dir, config=DEFAULT_SUBTITLE_CONFIG, font_path=None) -> Path` → `subtitles.ass`
- `media.background.compose_background(intro_path, loop_path, timing, attempt_dir, config=DEFAULT_BACKGROUND_CONFIG) -> Path` → `background.mp4` (intro/loop 해상도 일치 강제)
- `media.compose.compose_video_audio(background_path, voice_path, bgm_path, thumbnail_path, attempt_dir, config=DEFAULT_COMPOSE_CONFIG) -> Path` → `final.mp4`
  - **자막 인자 없음 → Task 1에서 subtitle_path/font 추가 필요**
- `media.audio.build_compose_cmd(...)` 필터그래프: 썸네일 오버레이(첫 프레임) + BGM afade + amix. **자막 필터 없음**
- `media.validation.validate_final_artifacts(attempt_dir) -> None` — `final.mp4`(1080×1920/30fps/영상+오디오/≥MIN), `subtitles.ass`(존재), `timing.json`(존재) 검사. `REQUIRED_WIDTH/HEIGHT/FPS` 상수 사용

### ⚠️ blocker — libass 포함 ffmpeg 필요 (2026-06-27 발견)

자막 번인(`ass` 필터)은 ffmpeg가 **libass와 함께 빌드**돼야 동작한다. 현재 dev 환경의
Homebrew ffmpeg 8.1.2는 `--enable-libass` 없이 빌드되어 `ass`/`subtitles` 필터가 없다
(`Filter not found`). 영향 범위:
- **dev 검증(Task 5)**: libass ffmpeg 없이는 실제 번인 영상 확인 불가
- **패키징**: `resources/ffmpeg/`에 번들할 ffmpeg도 libass 포함 정적 빌드여야 함
  (현재 비어 있음; macOS Homebrew ffmpeg는 동적 링크라 번들 불가)

조달 옵션: ① `homebrew-ffmpeg/ffmpeg` 탭 `--with-libass`(소스 컴파일, 수십 분),
② 신뢰 가능한 libass 포함 정적 ffmpeg 빌드 확보(패키징용). 사용자 결정 필요.
Task 3(orchestrator 배선)·Task 4는 mock으로 단위 구현·검증 가능하나, Task 5 실제
검증은 이 blocker 해소 후 가능.
