# Story 2.4: 음성 모델 성능 기준 확정

Status: done

## Story

As a 개발 담당자,
I want 기준 장비에서 음성 모델 설정을 비교하고 싶다,
so that 정확도와 처리 시간의 재현 가능한 기본값을 선택할 수 있다.

## Acceptance Criteria

1. 동일 한국어 샘플과 Windows/macOS 기준 장비에서 모델 크기·양자화·CPU thread 조합의 타이밍 정확도, 처리 시간, 메모리를 기록한다.
2. 선택 값을 typed runtime 설정과 회귀 테스트 기준으로 고정한다.

## Tasks / Subtasks

- [x] 라이선스·개인정보 안전한 대표 한국어 benchmark corpus와 정답 timing을 고정한다. (AC: 1)
- [x] 모델/compute type/thread 조합을 동일 조건에서 반복 측정하는 harness를 만든다. (AC: 1)
- [x] timing 오차, wall time, peak memory, 하드웨어·버전을 구조화 보고서로 기록한다. (AC: 1)
- [x] Windows x64와 macOS arm64 결과로 기본 설정을 선택하고 typed config에 고정한다. (AC: 2)
- [x] 허용 정확도·성능 회귀 기준과 오프라인 model availability 테스트를 추가한다. (AC: 2)

## Dev Notes

- 서로 다른 플랫폼 결과를 한 수치로 뭉개지 않는다. CPU·RAM·OS·runtime·model checksum을 기록한다.
- 샘플이 적어 통계적 결론을 과장하지 않는다. 최소 반복 횟수와 median/p95를 명시한다.
- 기준 미달 시 모델을 임의 교체하지 말고 결과와 trade-off를 보고한다. 모델 자산 변경은 패키지 크기·라이선스에 영향이 있다.
