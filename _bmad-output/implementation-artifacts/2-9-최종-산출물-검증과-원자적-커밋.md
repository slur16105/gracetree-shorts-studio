# Story 2.9: 최종 산출물 검증과 원자적 커밋

Status: ready-for-dev

## Story

As a 개인 제작자,
I want 검증된 결과만 최종 폴더에 저장되기를 원한다,
so that 불완전한 영상이 완료 결과로 남지 않는다.

## Acceptance Criteria

1. 임시 결과의 `1080x1920`, 9:16, 30fps MP4와 필수 stream·산출물 존재를 검증하고 그 전에는 final 경로를 변경하지 않는다.
2. 검증과 DB transaction 성공 시 `final.mp4`, `subtitles.ass`, `timing.json`, `render_log.txt`를 원자적으로 저장하고 완료 상태를 기록한 뒤 temp를 정리하고 단 하나의 completed를 보낸다.

## Tasks / Subtasks

- [ ] final artifact validator에 정확한 영상·stream·파일·비어 있지 않음 조건을 구현한다. (AC: 1)
- [ ] attempt temp에서 output pending 디렉터리로 staging하고 같은 파일시스템의 atomic rename 전략을 구현한다. (AC: 1, 2)
- [ ] DB 완료 기록과 파일 교체의 실패 보상 순서를 명시하고 트랜잭션으로 상태를 커밋한다. (AC: 2)
- [ ] 성공 로그 복사, temp 정리, terminal event 단일 발행을 구현한다. (AC: 2)
- [ ] 단계별 crash/failure injection으로 기존 output·DB 보존과 orphan 정리를 검증한다. (AC: 1, 2)

## Dev Notes

- “DB와 파일의 완전한 단일 원자성”을 가정하지 않는다. staging, rename, DB transaction, 보상·startup reconciliation 순서를 명시한다.
- final 경로에 FFmpeg가 직접 쓰면 안 된다. 검증된 temp 결과만 commit service가 이동한다.
- completed는 파일과 DB 완료 상태가 모두 확인된 뒤 한 번만 발행한다.
