# Story 2.3: 로컬 음성 정렬과 타이밍 생성

Status: ready-for-dev

## Story

As a 개인 제작자,
I want 음성과 스크립트를 로컬에서 정렬하고 싶다,
so that 수동으로 자막 시점을 맞추지 않아도 된다.

## Acceptance Criteria

1. 구조화 스크립트와 음성을 로컬 faster-whisper로 분석해 외부 전송 없이 앞 무음을 정리하고 영상 시작 약 2초 후 음성이 시작되도록 계산하며 원본을 변경하지 않는다.
2. 기도 시작점이 하나면 블록 순서와 ±0.75초 목표를 검증할 `timing.json`을 만들고 후보가 없거나 복수이면 `PRAYER_BOUNDARY_AMBIGUOUS`로 렌더링을 중단한다.

## Tasks / Subtasks

- [ ] 패키지된 로컬 model resolver와 typed speech 설정을 구현한다. (AC: 1)
- [ ] 음성 사본/임시 변환, 선행 무음 탐지와 약 2초 offset 계산을 구현한다. (AC: 1)
- [ ] whisper segment를 Story 2.2 timing blocks와 결정적으로 정렬한다. (AC: 1, 2)
- [ ] 기도 첫 구절 후보 0/1/N 규칙과 안정적 오류 진단을 구현한다. (AC: 2)
- [ ] schema-valid `timing.json`을 attempt temp에 쓰고 후속 단계에 전달한다. (AC: 2)
- [ ] 네트워크 차단, 원본 hash 불변, 한국어 golden sample, 경계 모호성 테스트를 추가한다. (AC: 1, 2)

## Dev Notes

- faster-whisper는 새 런타임·프로덕션 의존성이다. Architecture 승인 기술이지만 정확한 버전·모델·compute_type 설치는 구현 시 dependency approval 규칙을 확인한다.
- 원본 음성을 trim하거나 덮어쓰지 않는다. 모든 변환은 attempt temp에서 수행한다.
- 모델 성능 기본값은 Story 2.4가 확정한다. 이 Story는 typed provisional 설정을 사용하되 숨은 fallback 모델 다운로드를 금지한다.
- 모델과 입력을 외부로 전송하지 않고 인터넷이 없어도 실패 원인이 결정적이어야 한다.
