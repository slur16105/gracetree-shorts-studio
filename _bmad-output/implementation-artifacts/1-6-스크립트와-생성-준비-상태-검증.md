# Story 1.6: 스크립트와 생성 준비 상태 검증

Status: review

## Story

As a 개인 제작자,
I want 등록된 입력이 실제 생성에 충분한지 즉시 확인하고 싶다,
so that 잘못된 입력으로 긴 렌더링을 시작하지 않을 수 있다.

## Acceptance Criteria

1. 스크립트에서 `[제목]`, `[말씀]`, `[기도]`의 존재와 내용을 검증하고 빈·읽기 불가 파일을 거부한다.
2. 파싱 중 `스크립트를 확인하고 있습니다…`를 표시하고 생성 버튼을 비활성화한다.
3. 유효한 `[제목]`의 줄바꿈을 공백으로 합친 한 줄 제목과 전체 제목 접근 정보를 제공한다.
4. 누락 구역과 추가 방법을 안내하며 기술 코드만 표시하지 않는다.
5. 썸네일·음성·스크립트·유효 BGM 소스의 충족 개수, 백분율, 막대와 `필수 입력 n/4`를 재계산한다.
6. 누락·충돌·미분류·유효성 오류가 있으면 생성 버튼을 막고 다음 동작을 표시한다.
7. 네 조건과 공통 리소스가 모두 유효하면 `준비 완료`, `100% · 필수 입력 4/4`와 활성 생성 버튼을 표시한다.
8. 준비 상태 변경을 과도하게 낭독하거나 색상만으로 전달하지 않는다.
9. 정상 흐름의 사용자 조작 시간이 5분 이내인지 사용성 테스트로 측정한다.

- [x] Job Summary, Readiness Progress, 비활성 사유와 다음 동작을 구현한다. (AC: 2~8)
- [x] 입력 변화 시 영향받은 검증만 재실행하되 stale 결과를 현재 작업에 적용하지 않는다. (AC: 2, 5~7)
- [x] parser fixture, readiness matrix, live region, 5분 사용성 경로 테스트를 추가한다. (AC: 1~9)

## Dev Notes

- 이 Story의 parser는 생성 준비 검증에 필요한 최소 구조를 제공한다. Story 2.2가 빈 줄·자막 블록·타이밍 계약을 완성한다.
- 준비율과 생성 진행률은 다른 개념이다. 준비율은 job-editor, 렌더링 진행률은 Story 2.1의 job-progress가 소유한다.
- 작업별 BGM 또는 유효한 기본 BGM 중 하나가 BGM 조건을 충족한다. 공통 영상·폰트는 백분율 4개와 별도의 생성 gate다.
- 비동기 파싱 응답은 작업 ID와 입력 버전을 비교해 오래된 결과를 폐기한다.
