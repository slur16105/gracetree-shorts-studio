# Story 2.1: 최소 생성 수직 슬라이스와 진행 상태

Status: review

## Story

As a 개인 제작자,
I want 준비된 샘플 작업을 실제 엔진으로 끝까지 실행하고 싶다,
so that 생성 계약과 진행 화면이 조기에 검증된 기반 위에서 확장될 수 있다.

## Acceptance Criteria

1. 준비 완료 작업은 실행 스냅샷과 `attempt`를 만들고 실제 `start → stage_started → progress → completed` 흐름을 수행하며 필요한 상태·오류 필드를 순차 마이그레이션으로 추가한다.
2. 고정 샘플로 `temp/attempts/<attemptId>/vertical-slice.mp4`를 만들고 0바이트가 아니며 ffprobe 영상 스트림·양수 duration 검증 후에만 completed·100%와 파일명을 표시한다.
3. 현재 jobId와 허용 전이에 맞는 이벤트만 적용하고 단계·단조 증가 진행률을 1초 이내 표시하며 완료 전 100%를 금지하고 단계/10% 단위만 알린다.
4. 실행 중 입력·리소스 편집이 스냅샷을 바꾸지 않고 새 실행을 차단하며 다른 jobId·잘못된 전이는 무시·기록한다.

- [x] Zustand job-progress, bottom status, 실행 잠금과 live region throttle을 구현한다. (AC: 3, 4)
- [x] migration·계약·상태 머신·프로세스·E2E 수직 슬라이스 테스트를 추가한다. (AC: 1~4)

## Dev Notes

- 승인된 Change Proposal의 실제 MP4 검증은 필수다. timeout만 기다렸다 completed를 보내는 stub은 불가하다.
- 단 하나의 활성 attempt만 허용한다. terminal event는 completed/failed/cancelled 중 하나다.
- 완료 전 100% 금지, 진행률 역행 금지, jobId 불일치는 UI에 적용하지 않는다.
- 진단 산출물은 최종 결과가 아니며 `temp/attempts`에 남긴다. Story 2.9가 최종 커밋을 소유한다.
