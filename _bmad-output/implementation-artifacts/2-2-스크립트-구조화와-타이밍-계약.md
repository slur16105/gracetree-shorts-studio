# Story 2.2: 스크립트 구조화와 타이밍 계약

Status: review

## Story

As a 개인 제작자,
I want 스크립트 구조와 자막 블록이 정확히 해석되기를 원한다,
so that 음성 분석과 자막 생성이 같은 결정적 입력을 사용할 수 있다.

## Acceptance Criteria

1. 유효 스크립트에서 `[제목]`, `[말씀]`, `[기도]`와 기도 구역 빈 줄·줄바꿈을 구조화하고 정규화된 타이밍 입력 계약과 진단을 만든다.
2. 구역 누락·빈 내용은 결정적 `SCRIPT_INVALID`이며 고정 fixture 반복 실행 결과가 동일하다.

## Tasks / Subtasks

- [x] Story 1.6 validator를 확장해 parser AST와 subtitle/timing block DTO를 정의한다. (AC: 1, 2)
- [x] 원문을 변경하지 않고 newline·BOM·공백 정규화 규칙을 명시한다. (AC: 1)
- [x] 파싱 진단과 `SCRIPT_INVALID`를 공유 오류 계약에 연결한다. (AC: 2)
- [x] parser 결과를 attempt 실행 스냅샷과 후속 speech/subtitle 단계에 전달한다. (AC: 1)
- [x] 한국어·CRLF/LF·빈 줄·중복/누락 구역의 golden fixture 테스트를 추가한다. (AC: 1, 2)

## Dev Notes

- 완전 자동 문장 분할은 MVP 비목표다. 사용자가 지정한 기도 구역의 빈 줄·줄바꿈을 보존한다.
- parser는 UI 문구가 아닌 구조화 데이터와 안정적 진단을 반환한다.
- 같은 bytes와 설정은 같은 구조·오류를 만들어야 한다. 현재 시각·플랫폼 개행에 의존하지 않는다.
