# GraceTree Shorts Studio 기술·UX 입력 메모

이 문서는 PRD Discovery 중 제시된 기술적 구현 아이디어와 후속 아키텍처·UX 설계 입력을 보존한다. 아직 확정된 아키텍처 결정은 아니다.

## 사용자 제안 기술 방향

- 프론트엔드 후보: Next.js
- 입력 파일과 생성 파일은 외부 서버가 아닌 로컬 저장소에 보관
- PC 전용 화면으로 설계하며 모바일 대응은 제외
- 작업별 입력 파일은 선택한 게시 날짜의 애플리케이션 관리 폴더에 복사해 보관
- `intro_6s.mp4`, `prayer_loop_18s.mp4`, 기본 BGM 및 폰트는 설정에서 관리하는 공통 리소스
- 작업별 BGM이 등록되면 기본 BGM 대신 해당 작업에만 적용

## 사용자 제안 화면 방향

- 음악 재생 애플리케이션과 유사한 한 화면 중심 레이아웃
- 왼쪽의 앨범 아트 크기 영역을 파일 드래그앤드롭 영역으로 사용
- 여러 파일을 한 번에 드롭하면 파일 종류를 자동 판별하고 필요한 슬롯에 분류
- 메인 화면에서 바로 생성 실행
- 필요한 파일이 누락되면 실행 전에 누락 항목 안내
- 생성 중에는 제작 상태와 진행률을 표시
- 완료 시 모달로 알림
- 최근 작업 또는 마지막 생성 결과를 메인 영역에 표시
- 오른쪽에 제목과 생성일을 포함한 완료 작업 목록 표시
- 생성 완료 후 결과 폴더 열기 제공

## 후속 설계에서 검토할 기술 쟁점

- 일반 브라우저 기반 Next.js 앱의 로컬 파일 시스템 접근 제약
- 로컬 백엔드와 웹 UI 조합, 또는 데스크톱 패키징 방식 비교
- 대용량 영상 파일을 브라우저 메모리에 적재하지 않고 처리하는 방식
- 파일명·확장자·메타데이터를 이용한 자동 분류와 수동 교정 방식
- FFmpeg/Whisper 작업 진행률을 UI에 전달하는 방식
- 생성 이력의 로컬 저장 형식과 결과 파일 경로 관리
- 게시 날짜를 작업의 고유 키로 사용할지 내부 작업 ID와 함께 사용할지 결정
- 입력 파일을 날짜별 관리 폴더에 안전하게 복사하고 중복·교체를 처리하는 방식
- 공통 리소스 교체 시 파일을 애플리케이션 관리 폴더로 복사할지 원본 경로만 참조할지 결정

## 외부 사용자 제공 방식

### 후보 A: 설치형 로컬 애플리케이션

- 사용자가 PC에 애플리케이션을 설치하고 FFmpeg·Whisper 작업을 자신의 컴퓨터에서 실행
- 입력 파일과 결과 파일을 사용자 PC에 저장
- 핵심 영상 생성 기능에는 별도 호스팅이 필수적이지 않음
- Next.js 기반 UI를 유지하려면 데스크톱 셸 또는 로컬 서버 패키징 방식 검토
- 자동 업데이트, 라이선스, 로그인, 오류 수집을 추가할 때만 소규모 호스팅이 필요할 수 있음

### 후보 B: 호스팅형 웹 서비스

- Next.js 웹 UI, 사용자 인증, 데이터베이스, 파일 저장소, 작업 큐, 영상 렌더링 워커가 필요
- 대용량 입력 파일은 프론트엔드 호스트의 API를 경유하지 않고 객체 저장소에 직접 업로드하는 구조 권장
- FFmpeg·Whisper는 장시간 CPU·메모리 작업이므로 전용 컨테이너 또는 워커 서비스에서 실행
- Vercel은 UI와 경량 API, Railway는 초기 렌더링 워커 후보로 사용할 수 있음
- 다중 사용자 확장 시 Railway 단일 볼륨보다 S3·R2·Vercel Blob 같은 객체 저장소에 원본과 결과를 보관하는 구조 권장
- 사용자별 격리, 저장 기간, 비용 제한, 작업 대기열, 삭제 정책과 개인정보 정책이 추가로 필요

### 현재 권장 경로

1. 개인용 설치형 로컬 MVP로 영상 품질과 작업 흐름 검증
2. 소수 사용자에게 설치 파일을 배포해 다른 PC 환경에서 검증
3. 사용 수요와 운영 비용이 확인된 뒤 호스팅형 서비스용 별도 아키텍처 설계

## 확정된 제품 구조 원칙

- MVP 배포 형태는 개인용 설치형 로컬 애플리케이션이다.
- UI, 작업 관리, 영상 생성 엔진의 책임을 분리한다.
- 영상 생성 엔진은 로컬 파일 경로와 설정을 입력받아 진행 이벤트와 결과를 반환하는 독립 경계로 설계한다.
- 현재는 로컬 저장과 단일 사용자만 구현하되, 향후 저장소·작업 실행기·사용자 경계를 교체할 수 있도록 한다.
- 미래 SaaS를 이유로 현재 단계에서 인증, 결제, 객체 저장소 또는 분산 작업 큐를 미리 구현하지 않는다.
