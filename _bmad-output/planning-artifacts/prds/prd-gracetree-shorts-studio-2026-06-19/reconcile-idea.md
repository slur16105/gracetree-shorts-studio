# 입력 대조 — docs/idea.md

## 대조 결과

`docs/idea.md`의 제품 목적, 입력 구조, 영상 규격, 구간 처리, 자막, Whisper, FFmpeg, 산출물 및 MVP 제외 범위를 PRD와 대조했다.

## 확인된 반영 사항

- CapCut 반복 편집을 자동 MP4 생성 파이프라인으로 전환
- 1080x1920, 9:16, 30fps MP4 출력
- 썸네일 첫 프레임 삽입
- 제목·말씀 영상 길이 조절
- 기도 영상 반복과 0.5초 크로스페이드
- 음성 앞 무음 정리와 시작 여백
- 음성 분석 기반 타이밍 및 신뢰도 부족 시 중단
- ASS 자막과 구역별 스타일
- 배경음악 2초 페이드인·페이드아웃
- 음성 종료 후 약 3초 종료 여백
- `final.mp4`, `subtitles.ass`, `timing.json`, `render_log.txt`
- YouTube 업로드, 음량 자동 보정, 자동 자막 분할 제외

## Discovery 중 변경된 사항

- 작업별 BGM은 필수 입력에서 선택 입력으로 변경하고 설정의 기본 BGM을 사용한다.
- 입력 파일은 선택한 게시 날짜별 작업 폴더에 복사해 보관한다.
- UI, 날짜별 작업 관리, 완료 목록, 취소, 다시 생성 및 설정 화면을 MVP에 추가한다.
- 개인용 설치형 로컬 앱을 우선하되 UI와 영상 생성 엔진을 분리한다.

## 검토 중 보완한 누락

- 최종 출력의 30fps 기준을 FR-16에 명시했다.
- 마지막 “아멘” 자막 유지·페이드 조건을 FR-13에 명시했다.
- 배경음악 페이드 시간을 FR-15에 명시했다.

## 결론

원본 아이디어의 핵심 제작 규칙은 보존됐다. Discovery 중 추가된 UI·날짜 관리 요구사항은 원본 목표를 변경하지 않고 반복 사용성과 데이터 관리를 확장한다.
