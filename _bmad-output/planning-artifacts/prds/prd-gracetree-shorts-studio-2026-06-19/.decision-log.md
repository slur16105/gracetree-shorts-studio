# PRD 결정 로그

## 입력 자료

- `docs/idea.md` — 말씀기도 쇼츠 자동 제작 시스템 아이디어 문서

## 결정

- 초기 사용자는 제작자 본인 한 명이다.
- 먼저 실제 작업에 사용하는 로컬 도구로 검증한다.
- 사용 결과에서 상품화 가능성이 확인되면 서비스 확장 또는 재설계를 검토한다.
- 핵심 성공 조건은 필요한 입력 파일로부터 영상이 자동 생성되는 것이다.
- 1차 사용 환경은 PC이며 모바일 대응은 고려하지 않는다.
- 주요 작업 흐름은 파일 등록 → 입력 검증 → 생성 실행 → 진행률 확인 → 완료 및 결과 확인이다.
- 메인 화면에서 파일 등록과 생성을 바로 수행할 수 있어야 한다.
- 완료된 작업 목록과 최근 작업 결과를 확인할 수 있어야 한다.
- 입력 및 생성 파일은 당분간 외부 서버에 업로드하지 않고 로컬에 보관한다.
- PRD 작성 방식은 Fast path로 진행한다.
- 제품 중요도는 개인 실사용 MVP로 시작하되 향후 상품화 가능성을 검증하는 수준으로 설정한다.
- MVP UI를 영상 생성 엔진과 함께 개발한다.
- MVP는 한 번에 쇼츠 한 개를 생성하고, 여러 작업 일괄 처리는 후속 단계에서 검토한다.
- 여러 입력 파일은 한 번의 드래그앤드롭으로 등록한다.
- 시스템은 입력 파일을 자동 분류하되 누락·중복·분류 충돌을 안내한다.
- 오류 발생 시 원인과 권장 다음 동작을 모달로 안내한다.
- 완료 목록에서는 애플리케이션 내부 재생·삭제 없이 결과 폴더 열기만 제공한다.
- 주요 성공 기준은 사용자 조작 시간 5분 이내와 추가 편집 없는 자동 생성이다.
- 사용자가 준비하는 6개 자료는 6일치 쇼츠 작업 세트다.
- MVP는 6일치를 일괄 처리하지 않고 한 번에 쇼츠 한 개를 생성한다.
- MVP에 실행 중 작업 취소 기능을 포함한다.
- 품질 성공 기준은 최소 10개 작업 중 8개 이상이 추가 편집 없이 사용 가능한 것이다.
- 입력 파일은 사용자가 선택한 게시 날짜별 작업 폴더에 복사해 보관한다.
- 드래그앤드롭한 출처 파일은 변경하거나 삭제하지 않는다.
- 처리에 추가로 필요한 임시 파일은 작업 종료 후 정리하고 로그는 보존한다.
- 공통 리소스인 제목·말씀 영상, 기도 영상, 기본 배경음악 및 폰트는 MVP 설정 화면에서 관리한다.
- 기본 배경음악을 자동 적용하고, 작업별 배경음악 등록 시 해당 작업에만 대체 적용한다.
- 사용자는 생성 전에 게시 날짜를 선택한다.
- 완료 목록에는 게시 날짜, 실제 생성일을 표시한다.
- 완료 영상이 있는 게시 날짜에는 중복 생성을 막고 기존 결과를 안내한다.
- 완료된 게시 날짜는 확인 절차를 거쳐 다시 생성할 수 있다.
- 다시 생성 성공 전까지 기존 완료 결과를 유지하고, 성공한 경우에만 교체한다.
- MVP는 개인용 설치형 로컬 애플리케이션으로 개발한다.
- UI와 영상 생성 엔진을 분리해 향후 설치형 배포 확대 또는 호스팅 서비스 전환 가능성을 보존한다.
- 확장 가능성을 위해 MVP에 인증, 결제, 클라우드 저장소와 작업 큐를 미리 구현하지 않는다.

## 열린 항목

- 로컬 실행 및 배포 형태
- 완료 목록의 보존 기간과 삭제 방식

## 검토된 후속 제품 경로

- 설치형 로컬 앱은 핵심 영상 생성 기능에 별도 호스팅이 필수적이지 않다.
- 호스팅형 서비스는 웹 UI 외에 객체 저장소, 데이터베이스, 작업 큐, 렌더링 워커와 사용자 격리가 필요하다.
- 현재 권장안은 로컬 MVP 검증 후 설치형 배포를 먼저 검토하고, SaaS는 별도 확장 단계로 판단하는 것이다.

## 결정 로그 감사

- 제품 목적, 사용자, MVP 범위, 성공 지표와 파일 보관 정책은 `prd.md`에 반영됐다.
- Next.js, Python, FFmpeg, Whisper, 데스크톱 패키징과 향후 호스팅 후보는 `addendum.md`에 반영됐다.
- `docs/idea.md`의 제작 규칙과 Discovery 변경 사항은 `reconcile-idea.md`에서 대조됐다.
- 기준 PC의 렌더링 목표는 아키텍처 단계, 결과 폴더 외부 삭제 동작은 UX 단계에서 결정한다.
- 감사 결과 누락되거나 폐기된 결정은 없다.

## Fast path 가정

- 진행률은 생성 단계 기반 추정치로 표시한다.
- 자동 재시도는 제공하지 않는다.
- 취소 시 임시 파일은 정리하고 로그는 보존한다.
- 5분 성공 기준은 사용자 조작 시간이며 실제 렌더링 시간은 별도로 측정한다.
- 음성 분석은 로컬 실행을 기본으로 한다.
- 7일 연속 실사용으로 반복 안정성을 검증한다.

## 최종 검토 결정

- 음성 녹음은 `voice.*`, 작업별 배경음악은 `bgm.*` 파일명 규칙으로 우선 분류한다.
- 무편집 성공은 출력 규격, 자막 표시·싱크, 배경 영상 연속성, 음성·배경음악 합성을 모두 충족하는 것으로 정의한다.
- 완료 목록은 게시 날짜 내림차순으로 표시한다.
- 비정상 종료 시 입력과 기존 결과는 유지하고 미완료 작업의 재실행을 안내하며 중단 지점 자동 복구는 하지 않는다.
- 렌더링 시간은 작업별 로그에 기록하고 PC 기준 성능 목표는 아키텍처 단계에서 확정한다.
- 결과 폴더 외부 삭제 처리와 상품화 판단의 추가 신호는 MVP 구현을 막지 않는 후속 결정으로 둔다.
