## Foundation

- **폼팩터**: 데스크톱 단일 창(Electron + React). 모바일·반응형 레이아웃 없음. 권장 최소 콘텐츠 1180×720, 최소 동작 보장.
- **UI 시스템**: 별도 디자인 시스템·UI 라이브러리를 도입하지 않는다. 기존 CSS 모듈(`*.module.css`) + `tokens.css`의 CSS 커스텀 프로퍼티가 유일한 스타일 레이어다. 마크업·클래스 구조는 유지하고 토큰만 갱신한다.
- **시각 정체성**: 전적으로 [DESIGN.md](./DESIGN.md)를 따른다. 색·타이포·간격·모양·컴포넌트 표면 값은 모두 그곳의 토큰을 `{path.to.token}`으로 참조한다.
- **성격**: 로컬·오프라인 제작 도구. 한 번 설정한 공통 리소스를 두고 날마다 새 스크립트로 쇼츠를 반복 생성한다. 핵심 가치는 "지금 생성 가능한 상태인가"와 "방금 만든 결과가 어디 있는가"를 즉시 읽히게 하는 것.

## Information Architecture

세 개의 1차 화면을 전역 사이드바로 전환한다. 중복 목록 전용 화면을 만들지 않는다.

- **홈(Home)** — 헤더 + 2-pane + 푸터(레이아웃 A).
  - 헤더: eyebrow + 화면 제목(`영상 작업`)만. 날짜·생성 버튼은 두지 않는다.
  - 왼쪽 입력 영역: 최상단 한 줄에 **Date Picker(좌) + 영상 생성 버튼(우)** → 그 아래 Input Drop Zone(파일 슬롯). 워크스페이스 라벨·인라인 Job Summary 카드는 없다.
  - 오른쪽 사이드 열: **입력 준비 카드(Readiness)** → **완료 목록(Completion Row)** + 새로고침.
  - 푸터: `현재 작업: <제목>` + 생성 진행률 바(완료 목록 열 폭에 정렬).
- **사용 가이드(Guide)** — 내부 섹션 내비게이션 + 가이드 카드(3단계 시작, 파일 규칙, 문제 해결, 저장 위치).
- **공통 리소스 설정(Resource Settings)** — 모달 1단계. 제목·말씀 영상 / 기도 반복 영상 / 기본 배경음악 / 자막 폰트 각 행의 현재 파일·유효 상태·교체.

**서피스 폐합**: 입력 → 준비 → 생성 → 결과 확인이라는 단일 흐름이 홈 한 화면에서 끝난다. 가이드와 설정은 흐름을 보조하며, 어떤 동작도 새로운 최상위 화면을 추가로 요구하지 않는다.

## Voice and Tone

- 한국어 기본. 차분하고 군더더기 없는 도구의 말투. 종교적 수사를 전면에 두지 않는다.
- 날짜는 항상 `YYYY-MM-DD · Mon` 형식. 게시일과 실제 생성일을 라벨로 명확히 구분한다.
- 상태 문구는 사실 + 다음 동작을 함께 담는다. 예: "기도 반복 영상이 없습니다 — 공통 리소스 설정에서 추가하세요."
- 오류는 기술 코드를 노출하지 않고 사용자가 취할 다음 동작을 한 문장으로 제시한다. (코드/로그는 접근성 이름·상세 토글 뒤에 둔다.)
- eyebrow 같은 영문 대문자 안내는 화면당 하나 이하.

## Component Patterns (behavioral)

시각 스펙은 DESIGN.md.Components에 있다. 여기서는 동작만 규정한다.

- **Input Drop Zone / File Slot** — 드래그앤드롭과 파일 선택을 모두 받는다. 등록된 파일은 역할이 자동 분류되며, 상태(ready/conflict/unclassified/missing/invalid)는 색 + 아이콘 + 문구 3중으로 전달한다. per-row 액션은 두 가지로 분리한다:
  - **삭제(✕)** — 파일명 바로 우측 인라인. 그 파일을 제거한다.
  - **다시 선택** — 행 우측. 평소 상태(`✓ 준비됨`)를 보이다가 hover/`:focus-within`에서 노출. 전역 "파일 다시 선택" 버튼은 없앤다.
  - 액션 노출은 마우스 hover와 키보드 포커스 모두에서 동작하고 Tab 순서에 포함한다. 액션 세트는 상태에 따른다(ready→다시 선택·삭제, 빈/누락→선택).
  - 삭제(✕)는 **확인 한 번**(작은 `[취소] [삭제]` 확인) 후 제거한다.
- **Readiness Progress** — 생성 *전* 입력 충족률만 표현한다(렌더링 진행률과 분리). 막대는 항상 백분율과 충족 개수(`100% · 필수 입력 4/4`)를 동반한다. 미충족 시 다음 동작 또는 "설정 열기" 링크를 노출한다.
- **Job Summary (제목)** — `script.txt`에서 파싱한 제목은 인라인 카드 대신 **푸터의 `현재 작업: <제목>`** 으로 표시한다. 파싱 중·오류 상태를 구분하고, 오류는 해결 문장을 제시한다.
- **Date Picker** — 게시일 선택. 닫힌 트리거는 `YYYY-MM-DD · Sat`, 팝오버는 달력. 키보드로 열고·이동·선택·닫기 가능. 선택일만 강조.
- **Completion Row** — 선행 아이콘 없이 밀도 높은 행. 게시일 + 제목을 함께 보이고, 클릭으로 선택, 다시 클릭/Enter로 결과 폴더 열기. 결과 폴더 부재 시 인라인 경고(`결과 폴더 없음`)로 표시하고 열기 동작을 비활성화한다.
- **Primary/Secondary Button** — 주요 동작(영상 생성)은 게시일 오른쪽에 두고 입력 준비 100%(4/4) 전에는 비활성, 준비 완료 시 활성화한다. 보조 동작은 항상 취소 가능 경로를 제공한다.
- **Dialogs (Result / Failed / Cancel / Regenerate / Settings)** — 한 번에 하나만, 핵심 정보만. 제목은 한 줄, 다음 동작 버튼은 최소(2개 내외). **결과 경로·기술 details·반복 포맷 안내 같은 노이즈는 노출하지 않는다**(기술 로그는 `로그 폴더`, 경로는 `폴더 열기` 뒤로). 파괴적 확인(재생성·취소)은 danger 강조.

## State Patterns

- **빈 상태(Empty)** — 홈 작업 영역과 완료 목록 각각의 빈 상태를 점선 영역 + 안내 문구로 제시한다. 무엇을 하면 채워지는지 한 문장.
- **준비(Readiness)** — 미충족 → 충족(`{colors.progress}`). 준비율은 작업 영역에만, 생성 진행률은 상태 바에만 둔다(이중 표기 금지).
- **진행(Running)** — 상태 바 점이 `{colors.progress}` 펄스. 단계·백분율을 텍스트로 동반. 취소 가능 구간을 명확히.
- **성공(Success)** — Result Dialog + 완료 목록에 행 추가. 성공 신호는 `{colors.progress}`(녹색 아님).
- **경고(Attention)** — 확인 필요 입력·결과 폴더 부재 등은 `{colors.warning}`. 차단이 아니라 주의.
- **실패(Failure)** — `{colors.danger}` + 다음 동작. 기술 로그는 직접 노출하지 않는다.
- **비활성(Disabled)** — `{colors.text-disabled}` 또는 낮은 불투명도. 비활성이 정보 전달의 유일한 수단이 되지 않게 보조 문구를 둔다.

## Interaction Primitives

- **포커스**: 모든 인터랙티브 요소는 보이는 포커스 링(`{colors.focus}`)을 갖는다. 키보드만으로 전체 흐름(입력→날짜→생성→결과 열기)을 완주할 수 있어야 한다.
- **hover/active**: 표면을 한 단계 밝혀(`{colors.hover}`/`{colors.selected}`) 피드백한다. 색이 아니라 표면 변화로 hover를 표현한다.
- **모션**: 준비율·진행률 막대는 짧은 transition(약 0.3s)만. `prefers-reduced-motion: reduce`에서 transition·펄스를 끈다.
- **드래그**: 드롭존 전체가 드롭 타깃. 드래그 진입 시 경계·표면 변화로 명확히 신호한다.

## Accessibility Floor

- 본문 텍스트 대비 WCAG 2.2 AA(4.5:1) 이상. 인디고 채움 위 텍스트는 어두운 전경으로 대비 확보.
- 상태는 색 단독으로 전달하지 않는다 — 아이콘·문구를 항상 동반(ready/warning/danger 모두).
- 모든 동작 키보드 접근 가능, 포커스 순서가 시각 순서와 일치. 모달은 포커스 트랩 + Esc 닫기.
- 말줄임된 제목·파일명은 전체 값을 `title`/접근성 이름으로 제공.
- 스크린리더용 텍스트(`.srOnly`)로 상태 점·아이콘의 의미를 보강.

## Key Flows

### Flow 1 — 은혜교회 미디어 간사 지수의 아침 제작 (climax: 준비 완료 → 생성)

1. 지수가 앱을 연다. 홈 작업 영역은 빈 상태, 완료 목록엔 어제 만든 행들이 있다.
2. 오늘 게시일을 Date Picker에서 `2026-06-28 · Sun`으로 고른다.
3. 폴더에서 `script.txt`와 말씀·기도 영상을 드롭존에 끌어 놓는다. 슬롯이 자동 분류되고, 한 파일이 미분류로 떠 역할을 직접 고른다.
4. 푸터에 `현재 작업: <제목>`이 뜬다. 오른쪽 입력 준비 카드가 `75% · 3/4`에서 멈춘다 — 배경음악이 없다. 게시일 옆 `영상 생성` 버튼은 아직 비활성이다.
5. "설정 열기"로 공통 리소스 설정을 열어 기본 배경음악을 확인하고 닫는다. 준비율이 **`100% · 4/4`**로 차며 `{colors.progress}`로 채워진다. — **이 순간이 절정**: 게시일 오른쪽의 `영상 생성` 버튼이 활성화된다.
6. `영상 생성`을 누른다. 푸터 진행률 바가 0%부터 `{colors.progress}`로 차오르고, 상태 점이 펄스한다.
7. 완료되면 Result Dialog가 한 문장 요약과 `폴더 열기`를 제시한다. 완료 목록에 오늘 행이 추가된다.

### Flow 2 — 실패에서 회복

1. 생성 중 입력 하나가 손상돼 실패한다.
2. Failed Dialog가 `{colors.danger}`로 "자막 폰트를 읽지 못했습니다 — 설정에서 폰트를 교체한 뒤 다시 생성하세요"를 보여준다. 기술 로그는 표면에 없다.
3. 지수가 설정에서 폰트를 교체하고 다시 생성한다. 흐름이 Flow 1의 6단계로 복귀한다.
