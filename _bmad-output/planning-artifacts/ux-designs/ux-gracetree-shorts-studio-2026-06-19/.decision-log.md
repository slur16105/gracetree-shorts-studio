## 2026-06-19 — 디자인 방향 시안

- 비교 시안: `.working/directions-3.html`
- A `Studio Black`: YouTube Music과 가장 가까운 구조, 중간 정보 밀도, 분홍색 주요 강조와 녹색 준비·진행 상태
- B `Quiet Chapel`: 말씀기도의 차분한 정서를 강조, 넉넉한 여백, 따뜻한 금색 강조, 부드러운 한국어 안내
- C `Render Console`: 전문 제작 도구 성격, 높은 정보 밀도, 파일·검증·렌더링 진단 정보 우선
- 선택: A `Studio Black`
- 선택 이유로 해석되는 기준: 사용자가 처음 제시한 YouTube Music 화면과 가장 가까운 익숙한 구조
- 최종 설계 기준:
  - 검은색·짙은 회색 중심의 다크 인터페이스
  - 왼쪽 드래그앤드롭 작업 영역과 오른쪽 완료 목록의 명확한 분할
  - 하단 고정 상태 바에서 최근 작업과 렌더링 진행률 표시
  - 준비·성공 상태는 녹색 계열로 빠르게 식별
  - 주요 생성 동작은 배경과 강한 대비를 갖는 버튼으로 표시
  - 왼쪽 작업 영역 하단의 막대는 `입력 준비율`로 정의
  - 준비 상태 문구 아래에 `100% · 필수 입력 4/4`처럼 백분율과 충족 개수를 함께 표시
  - 영상 생성 진행률은 입력 준비율과 혼동되지 않도록 하단 고정 상태 바에 단계명과 백분율로 별도 표시
  - 작업 버튼 순서는 왼쪽 `파일 다시 선택`, 오른쪽 `영상 생성`; 주요 생성 동작을 오른쪽에 배치
  - 보조 버튼은 짙은 배경과 밝은 글자, 주요 버튼은 밝은 배경과 어두운 글자로 구분
  - 작업 제목은 등록된 `script.txt`의 `[제목]` 구역을 한 줄로 정리해 표시
  - 파일 등록 전 제목 위치에는 `새 쇼츠를 준비하세요`를 표시하고 `파일을 등록하면 스크립트 제목이 여기에 표시됩니다.`를 안내
  - 스크립트 등록 후 파싱 중에는 `스크립트를 확인하고 있습니다…`, 성공 후 실제 제목을 표시
  - `[제목]`이 누락되면 `제목을 확인할 수 없습니다`와 스크립트 수정 안내를 표시하고 생성 버튼을 비활성화
  - 게시 날짜 선택기를 누르면 A안과 동일한 검정·짙은 회색 배경의 달력 팝오버가 열림
  - 달력 요일 머리글은 `Sun Mon Tue Wed Thu Fri Sat`, 선택일은 A안의 분홍 강조색으로 표시
  - 메인 날짜 선택기 표기는 `YYYY-MM-DD · Sat` 형식
  - 완료 목록의 게시 날짜는 작업 폴더·파일명 기준과 동일한 `YYYY-MM-DD · Mon` 형식으로 표시
  - 실제 생성일은 게시 날짜 뒤에 별도 메타정보로 표시하며 게시일과 혼동하지 않음
  - 왼쪽 사이드바는 `홈`, `사용 가이드`, `공통 리소스 설정` 세 동작으로 구성
  - 완료 목록 전용 사이드바 버튼은 오른쪽 목록과 중복되므로 제거
  - `사용 가이드`에는 첫 영상 만들기, 파일명 규칙, 스크립트 예시, 오류 해결, 저장 위치, 앱 정보·버전을 제공
  - 제작자 소개는 작업에 직접 필요하지 않으므로 독립 메뉴로 두지 않고 사용 가이드의 `앱 정보` 하단에 짧은 크레딧으로 포함
  - 공통 리소스 설정은 한 단계 모달로 표시하며 제목·말씀 영상, 기도 반복 영상, 기본 배경음악, 자막 폰트를 한 화면에서 확인·교체
  - 설정 모달은 A안의 배경·테두리·버튼 대비를 그대로 사용하고 모달 위에 추가 확인 모달을 쌓지 않음
  - 사용 가이드는 모달이 아닌 독립 화면으로 제공하여 긴 설명과 예시를 읽기 쉽게 구성
  - 가이드 화면에서도 전역 사이드바를 그대로 유지하며 `?` 아이콘을 활성 상태로 표시
  - 가이드에서 집 아이콘을 누르면 입력 상태를 보존한 채 홈 제작 화면으로 돌아감
  - 가이드 왼쪽 내부 탐색: 첫 영상 만들기, 파일명 규칙, 스크립트 작성법, 오류 해결, 저장 위치, 앱 정보
  - 첫 영상 만들기는 게시 날짜 선택 → 파일 일괄 등록 → 준비율 확인 및 생성의 3단계로 설명
- 폐기 방향:
  - B의 감성적·종교적 시각 표현은 채택하지 않음
  - C의 개발자 콘솔식 영문 용어와 고밀도 진단 화면은 채택하지 않음
