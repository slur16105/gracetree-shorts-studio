# GraceTree Shorts Studio — Experience Spine

## Foundation

Windows와 macOS에서 동작하는 개인용 설치형 로컬 데스크톱 애플리케이션이다. 모바일·태블릿·브라우저 전용 화면은 지원하지 않는다. 기본 모드는 다크이며 시각 계약은 `DESIGN.md`가 소유한다.

한 번에 한 작업만 생성한다. 입력 파일과 결과는 로컬에 보관하고 네트워크 연결을 기본 경험으로 요구하지 않는다. UI 시스템과 데스크톱 패키징 방식은 아키텍처 단계에서 결정하되, 이 문서의 정보 구조와 상태 계약은 구현 기술과 무관하게 유지한다.

## Information Architecture

| Surface | Reached from | Purpose |
|---|---|---|
| 홈 제작 화면 | 앱 실행 / 사이드바 홈 | 게시 날짜 선택, 파일 등록·검증, 생성 실행, 최근 상태와 완료 목록 확인 |
| 사용 가이드 | 사이드바 `?` | 첫 영상 만들기, 파일명 규칙, 스크립트 작성법, 오류 해결, 저장 위치, 앱 정보 |
| 공통 리소스 설정 | 사이드바 설정 | 공통 영상, 기본 BGM, 폰트 상태 확인과 교체 |
| 달력 팝오버 | 홈 게시 날짜 선택기 | 게시 날짜 선택 |
| 완료·실패 결과 모달 | 생성 종료 | 결과 확인, 폴더 열기, 오류 해결 후 재시도 |
| 취소 확인 모달 | 생성 중 취소 | 실행 중인 작업 중단 여부 확인 |
| 다시 생성 확인 모달 | 완료 날짜에서 다시 생성 | 기존 결과 교체 조건 확인 |

홈은 전역 사이드바, 왼쪽 작업 영역, 오른쪽 완료 목록, 하단 상태 바로 구성한다. 사용 가이드는 전역 사이드바를 유지하는 독립 화면이다. 공통 리소스 설정은 한 단계 모달이다. 모달 중첩은 허용하지 않는다.

시각 참고: [Studio Black 홈·달력·설정·가이드](mockups/studio-black-reference.html#direction-a). 최종 문서와 충돌할 경우 `DESIGN.md`와 `EXPERIENCE.md`가 우선한다.

## Voice and Tone

간결하고 행동 중심으로 말한다. 영상 제작 지식이 없어도 다음에 무엇을 해야 하는지 알 수 있어야 한다.

| Do | Don't |
|---|---|
| `파일을 등록하면 스크립트 제목이 여기에 표시됩니다.` | `콘텐츠 메타데이터 대기 중` |
| `스크립트에 [제목] 구역을 추가한 뒤 다시 등록하세요.` | `SCRIPT_PARSE_ERROR` |
| `기도 시작점을 찾지 못했습니다. 스크립트와 음성의 첫 기도 문장을 확인하세요.` | `Whisper confidence low` |
| `결과 폴더를 찾을 수 없습니다.` | `경로 오류` |
| `영상 생성이 완료되었습니다.` | 과도한 축하 문구와 애니메이션 |

기술 세부정보는 `render_log.txt`에 기록하고 UI는 원인 범주와 사용자가 취할 행동을 우선한다.

## Component Patterns

| Component | Use | Behavioral rules |
|---|---|---|
| Sidebar Icon Button | 전역 | 홈, 사용 가이드, 설정. 현재 화면을 활성 표시한다. 설정은 모달을 열며 현재 화면을 바꾸지 않는다. |
| Drop Zone | 홈 | 여러 파일을 한 번에 받는다. 파일별 성공·거부를 독립 처리한다. 드롭 출처 파일은 변경하지 않는다. 클릭 파일 선택도 동일 동작을 제공한다. |
| File Slot | 홈 | 썸네일, 음성, BGM, 스크립트 역할을 표시한다. 제거·교체 시 준비 상태를 즉시 다시 계산한다. |
| Date Picker | 홈 | 기본값은 오늘. 선택 후 `YYYY-MM-DD · Sat`로 표시한다. 완료 날짜면 일반 생성을 막고 완료 항목·다시 생성 동작을 제시한다. |
| Calendar Popover | 홈 | 키보드 화살표로 날짜 이동, Enter로 선택, Esc로 닫는다. `Sun`~`Sat` 머리글을 사용한다. |
| Job Summary | 홈 | 등록 전 `새 쇼츠를 준비하세요`; 파싱 중 `스크립트를 확인하고 있습니다…`; 성공 후 `[제목]`을 한 줄로 표시한다. |
| Readiness Progress | 홈 | 준비 조건 4개(썸네일, 음성, 스크립트, 유효한 BGM 소스)를 `백분율 · 충족 개수`와 함께 표시한다. BGM 소스는 작업별 BGM 또는 정상인 기본 BGM 중 하나면 충족한다. 공통 영상·폰트가 유효하고 충돌이 없을 때만 생성 가능하다. |
| Primary Button | 홈·모달 | 홈에서는 `영상 생성`이며 버튼 묶음 오른쪽에 위치한다. 준비 전·실행 중에는 비활성화한다. |
| Secondary Button | 홈·모달 | `파일 다시 선택`, `취소`, `교체`. 주요 동작보다 먼저 배치한다. |
| Completion Row | 홈 오른쪽 | 제목, 게시일 `YYYY-MM-DD · Mon`, 실제 생성일, 결과 폴더 상태를 표시한다. 행 선택은 최근 작업 영역을 갱신한다. |
| Bottom Status Bar | 홈 | 유휴 시 최근 작업, 실행 중에는 생성 단계와 0~99% 진행률을 표시한다. 완료 직전에만 100%가 된다. |
| Resource Settings Dialog | 전역 | 공통 리소스 4종을 한 화면에서 확인·교체한다. 새 리소스는 이후 작업부터 적용한다. |
| Guide Navigation | 가이드 | 첫 영상 만들기, 파일명 규칙, 스크립트 작성법, 오류 해결, 저장 위치, 앱 정보를 전환한다. 홈 아이콘 복귀 시 입력 상태를 보존한다. |
| Guide Card | 가이드 | 첫 영상 만들기의 3단계와 파일 규칙을 짧은 단위로 설명한다. 장식용 카드나 마케팅 문구에 사용하지 않는다. |
| Result Dialog | 전역 | 성공은 결과 폴더 열기, 실패는 수정·설정 이동·재시도 중 관련 동작을 제공한다. 자동 재시도하지 않는다. |

## State Patterns

| State | Surface | Treatment |
|---|---|---|
| 첫 실행·이력 없음 | 홈 | `새 쇼츠를 준비하세요`와 드롭 안내. 완료 목록에는 첫 작업 안내를 표시한다. |
| 파일 등록 중 | 홈 | 파일별 처리 상태를 표시하고 중복 드롭을 막지 않는다. |
| 스크립트 파싱 중 | Job Summary | `스크립트를 확인하고 있습니다…`; 생성 버튼 비활성화. |
| 입력 일부 누락 | 홈 | 누락 슬롯과 형식을 표시한다. 준비율과 `필수 입력 n/4`를 갱신한다. |
| 입력 충돌·미분류 | 홈 | 해당 파일만 경고 상태로 두고 역할 선택 또는 제거 동작을 제공한다. 임의 배정하지 않는다. |
| 제목 누락 | Job Summary | `제목을 확인할 수 없습니다`; `[제목]` 추가 안내; 생성 버튼 비활성화. |
| 공통 리소스 누락 | 홈 | 생성 불가 사유와 `설정 열기` 동작을 제공한다. |
| 공통 리소스 교체 실패 | 설정 모달 | 새 파일 오류를 해당 행에 표시하고 기존 정상 리소스를 유지한다. 사용자는 모달을 닫거나 다른 리소스를 계속 교체할 수 있다. |
| 준비 완료 | 홈 | `준비 완료`, `100% · 필수 입력 4/4`, 활성 `영상 생성`. |
| 생성 중 | 홈 | 파일·날짜 변경을 현재 실행 작업에 적용하지 않는다. 하단에 단계와 진행률, 취소 동작을 표시한다. |
| 취소 요청 중 | 취소 확인 모달 | 취소 영향과 임시 파일 정리를 안내한다. |
| 생성 성공 | 결과 모달·홈 | 성공, 게시 날짜, 결과 위치를 표시하고 완료 목록을 갱신한다. |
| 생성 실패 | 결과 모달·홈 | 실패 단계, 이해 가능한 원인, 다음 동작을 표시한다. 완료 목록에 추가하지 않는다. |
| 앱·PC 비정상 종료 후 재실행 | 홈 | 날짜별 입력과 기존 완료 결과를 유지한다. 미완료 상태와 처음부터 다시 실행할 것을 안내한다. |
| 완료 날짜 선택 | 홈 | 기존 완료 상태와 폴더 열기를 표시하고 일반 생성 버튼을 비활성화한다. |
| 다시 생성 | 홈·확인 모달 | 성공 전까지 기존 결과를 유지한다. 실패·취소 시 기존 완료 기록을 보존한다. |
| 결과 폴더 없음 | Completion Row | 정상 완료처럼 열지 않고 `결과 폴더를 찾을 수 없습니다`를 표시한다. 기록 제거 정책은 후속 결정으로 남긴다. |
| 네트워크 없음 | 전역 | 정상 동작. 외부 연결을 요구하는 오류나 배너를 표시하지 않는다. |
| 좁은 창 | 전역 | 최소 창 크기를 안내하고 핵심 버튼·진행률을 자르지 않는다. 모바일 재배치는 하지 않는다. |

## Local File Safety

- 드롭한 원본 파일은 수정하거나 삭제하지 않는다.
- 등록 파일은 선택 게시 날짜의 앱 관리 폴더로 복사한다.
- 같은 날짜·역할 파일 교체는 명시적 확인 후 수행한다.
- 실패·취소·비정상 종료가 기존 완료 결과를 손상시키지 않는다.
- 임시 처리 파일은 종료 시 정리하고 진단 로그는 보존한다.
- 결과 폴더 열기는 운영체제 파일 탐색기를 사용한다. 앱 내부 재생·삭제는 제공하지 않는다.

## Interaction Primitives

- 마우스·트랙패드와 키보드를 동등하게 지원한다.
- `Tab`은 읽기 순서대로 이동하고 `Enter`/`Space`는 활성 컨트롤을 실행한다.
- `Esc`는 달력·모달을 닫는다. 실행 중 작업 자체를 즉시 취소하지 않는다.
- 드래그앤드롭의 모든 기능은 클릭 파일 선택으로도 가능해야 한다.
- 파괴적 동작인 취소와 다시 생성은 확인을 요구한다.
- 완료 목록은 게시 날짜 내림차순으로 정렬한다. 행 선택과 `폴더 열기` 버튼은 별도 포커스 대상이며 행 선택만으로 파일 탐색기를 열지 않는다.
- 자동 재시도, 타임라인 드래그 편집, 앱 내부 파일 삭제, 숨겨진 hover 전용 동작은 제공하지 않는다.

## Accessibility Floor

- WCAG 2.2 AA를 목표로 한다. 주요 텍스트 대비는 4.5:1 이상, 큰 텍스트와 비텍스트 UI는 3:1 이상.
- 모든 아이콘 버튼은 `홈`, `사용 가이드`, `공통 리소스 설정`처럼 접근성 이름을 갖는다.
- 색상만으로 정상·경고·실패를 표현하지 않고 아이콘과 상태 문구를 함께 쓴다.
- 키보드 포커스는 `{colors.focus-ring}`으로 명확히 보이며 배경 대비 3:1 이상을 목표로 한다.
- 진행률은 프로그레스바뿐 아니라 단계명, 백분율, 충족 개수 텍스트로 제공하고 변화는 과도하게 반복 낭독하지 않는다.
- 생성 진행률은 단계가 바뀌거나 10% 단위를 통과할 때만 polite live region으로 알린다. 연속 프레임 단위 갱신은 낭독하지 않는다.
- 파일 등록은 배치 처리가 끝난 뒤 성공·거부 개수를 한 번 알리고, 개별 오류 문구는 대응 File Slot과 연결한다.
- 달력은 접근성 grid 패턴을 사용한다. 화살표는 일 단위, Home/End는 현재 주의 첫날·마지막 날, PageUp/PageDown은 이전·다음 달, Enter/Space는 선택, Esc는 닫기다. 오늘과 선택일 상태를 각각 노출한다.
- 모달이 열리면 포커스를 내부로 이동하고 닫을 때 호출한 버튼으로 되돌린다.
- 운영체제의 모션 감소 설정을 존중한다. 진행 상태는 위치 이동이나 반복 펄스에 의존하지 않는다.
- Windows 디스플레이 배율과 macOS 확대 환경에서 200%까지 주요 작업이 가능해야 한다.

## Responsive & Platform

| Context | Behavior |
|---|---|
| Windows | 시스템 파일 선택기와 탐색기 열기 동작을 사용한다. 경로 구분자와 드라이브 문자를 UI 문구에 하드코딩하지 않는다. |
| macOS | 시스템 파일 선택기와 Finder 열기 동작을 사용한다. 창 닫기와 앱 종료를 구분한다. |
| 권장 크기 | 1180×720 이상. 홈의 좌우 분할과 하단 상태 바를 유지한다. |
| 최소 크기 미만 | 콘텐츠를 임의로 모바일 스택으로 바꾸지 않고 최소 창 크기 안내 또는 제한을 적용한다. |

## Inspiration & Anti-patterns

- **YouTube Music에서 차용:** 왼쪽의 큰 시각 영역과 오른쪽 목록, 하단의 지속 상태 영역, 어두운 표면 계층.
- **제품에 맞게 변경:** 앨범 영역은 파일 드롭·검증, 트랙 목록은 완료 작업, 플레이어는 렌더링 상태가 된다.
- **거부 — 음악 재생 제어의 직접 복제:** 재생·좋아요·셔플처럼 영상 제작에 의미 없는 아이콘은 사용하지 않는다.
- **거부 — 범용 영상 편집기:** 타임라인, 클립 자르기, 앱 내부 미리보기는 MVP에 포함하지 않는다.
- **거부 — 개발자 콘솔 UI:** 로그와 영문 코드보다 사용자가 취할 다음 동작을 우선한다.
- **거부 — 감성 장식 중심 UI:** 말씀기도의 정서는 콘텐츠가 전달하며 제작 도구는 중립적이고 명확하게 유지한다.

## Key Flows

### UJ-1. Slur가 게시 날짜에 맞춰 말씀기도 쇼츠를 생성한다.

1. Slur가 앱을 열면 홈 제작 화면과 오늘 게시 날짜가 표시된다.
2. 게시 날짜 선택기를 열어 `2026-06-20 · Sat`를 선택한다.
3. 썸네일, 음성, 스크립트를 드롭 영역에 한 번에 놓는다.
4. 시스템은 파일별 등록 결과와 자동 분류 슬롯을 표시하고 날짜별 작업 폴더에 복사한다.
5. `script.txt`를 해석하는 동안 `스크립트를 확인하고 있습니다…`를 표시한다.
6. 제목이 `타인의 슬픔을 바라볼 때 드리는 기도`로 바뀌고 `100% · 필수 입력 4/4`가 표시된다.
7. Slur가 오른쪽의 `영상 생성`을 누른다.
8. 하단 상태 바가 입력 검증, 음성 분석, 자막 생성, 영상 합성, 결과 저장 단계와 진행률을 표시한다.
9. **Climax:** 성공 모달에서 `영상 생성이 완료되었습니다`와 결과 위치가 보이고, 오른쪽 완료 목록에 `2026-06-20 · Sat` 항목이 추가된다.
10. Slur가 `결과 폴더 열기`를 눌러 운영체제 파일 탐색기에서 `final.mp4`를 확인한다.

실패 경로: 생성 실패 시 완료 목록에 추가하지 않고 실패 단계·원인·다음 동작을 표시한다. 기존 완료 결과는 변경하지 않는다.

### UJ-2. Slur가 누락되거나 잘못된 입력을 수정한다.

1. Slur가 파일을 드롭한다.
2. 시스템이 음성 파일 두 개를 같은 슬롯 후보로 감지하거나 `[제목]`이 없는 스크립트를 발견한다.
3. 해당 슬롯을 경고로 표시하고 준비율을 `75% · 필수 입력 3/4`처럼 갱신한다.
4. 생성 버튼은 비활성화되고 `중복된 음성 파일 중 하나를 제거하세요` 또는 `[제목] 구역을 추가하세요`가 표시된다.
5. Slur가 `파일 다시 선택` 또는 슬롯의 제거·교체 동작으로 문제를 수정한다.
6. 시스템이 영향을 받은 파일만 다시 검증한다.
7. **Climax:** 경고가 사라지고 제목·파일 슬롯·`100% · 필수 입력 4/4`가 동시에 정상 상태로 바뀌며 `영상 생성`이 활성화된다.

실패 경로: 읽을 수 없는 파일은 다른 정상 파일 등록을 막지 않고 해당 파일만 거부한다.

### UJ-3. Slur가 생성 실패를 복구한다.

1. Slur가 준비 완료 작업의 생성을 시작한다.
2. 음성 분석 중 기도 시작점을 하나로 결정하지 못해 작업이 중단된다.
3. 실패 모달은 `음성 분석 단계에서 기도 시작점을 찾지 못했습니다`를 표시한다.
4. 모달은 `스크립트와 음성의 첫 기도 문장을 확인하세요`라는 다음 행동과 `로그 폴더 열기`를 제공한다.
5. Slur가 파일을 수정해 다시 등록한다.
6. 자동 재시도 없이 Slur가 직접 `영상 생성`을 다시 누른다.
7. **Climax:** 재실행이 성공하고 기존 실패 기록이 완료 항목으로 오인되지 않은 채 새 완료 항목만 생성된다.

취소 경로: Slur가 생성 중 취소를 선택하면 확인 후 프로세스를 종료하고 임시 파일을 정리한다. 취소 작업은 완료 목록에 추가하지 않는다.
