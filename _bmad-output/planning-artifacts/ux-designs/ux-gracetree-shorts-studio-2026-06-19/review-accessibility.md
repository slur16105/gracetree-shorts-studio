# Accessibility Review — gracetree-shorts-studio

## Overall verdict

키보드 대체 입력, 모달 포커스 복귀, 텍스트 병행 진행률, 200% 확대 등 기본 접근성 계약은 적절하다. 색상 대비 두 곳과 동적 진행률·달력 의미론을 보완하면 WCAG 2.2 AA 구현 기준으로 사용할 수 있다.

## Findings

- **high — 달력 선택일 대비:** 흰색 `#FFFFFF`과 분홍색 `#FF2F62` 조합은 3.60:1이다. 작은 날짜 텍스트에 부족하다. *Fix:* 선택일 전경을 `#080909`로 변경한다.
- **medium — 컨트롤 경계 대비:** `#4C5151`과 `#080909`는 2.47:1이다. *Fix:* 강한 경계 토큰을 `#686E6E` 이상으로 조정한다.
- **medium — 진행률 공지:** 진행률 변화 낭독 빈도 규칙이 없다. *Fix:* 단계 변경과 의미 있는 10% 단위만 polite live region으로 알리고 매 프레임 갱신은 낭독하지 않는다.
- **medium — 달력 키보드 의미론:** 화살표·Enter·Esc는 정의됐지만 Home/End, PageUp/PageDown과 현재 선택·오늘 표기가 없다. *Fix:* grid 패턴과 표준 키 동작을 추가한다.
- **low — 완료 행 동작:** 행 선택과 `폴더 열기`의 포커스 대상 관계가 불명확하다. *Fix:* 행과 열기 버튼을 별도 포커스 대상으로 두고 행 선택만으로 파일 탐색기를 열지 않는다.
- **low — 드롭 상태 공지:** 파일별 등록 결과의 스크린리더 공지 방식이 없다. *Fix:* 등록 배치 종료 후 성공·거부 개수를 한 번 알리고 슬롯 오류는 해당 행에 연결한다.
